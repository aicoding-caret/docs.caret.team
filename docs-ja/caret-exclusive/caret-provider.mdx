# <BrandName />プロバイダー

<BrandName />プロバイダーはAPIキーなしで即座にAIコーディングアシスタントを使用できるようにする統合認証システムです。現在利用可能です。

## これは何ですか？

<BrandName />を初めて使用する多くの開発者が直面する最初の障壁は、まさにAPIキーの設定です。OpenAIやAnthropicのようなAIプロバイダーのAPIキーを発行し、支払い情報を登録し、使用量を管理するプロセスは煩わしいことがあります。

<BrandName />プロバイダーはこのような初期参入障壁を取り除きます。<BrandName />アカウント1つあれば、別途のAPIキー設定なしに即座にAIアシスタントを使用できます。

## Google Gemini優先サポート

<BrandName />プロバイダーの最初のサポートモデルは**Google Gemini**です。Geminiは強力なコーディング能力を持ちながら、相対的に手頃な価格で提供されるため、初心者と個人開発者に適しています。

## 無料クレジット

<BrandName />プロバイダーを初めて使用するすべてのユーザーに**無料クレジット**を提供します。これにより、実際のコスト負担なしに<BrandName />の機能を十分にテストできます。

## 画像生成ツール (<BrandName />アカウント)

<BrandName />アカウントでログインすると、画像生成ツールを利用できます。この機能は <BrandName /> トークンで認証され、生成結果は作業ディレクトリに自動保存されます。

### 仕組み

1. <BrandName /> に画像生成を依頼します (例: 「ダッシュボード用のイラストを作って」).
2. <BrandName /> が画像生成 API を呼び出して結果を作成します。
3. 生成物は作業ディレクトリの `.agents/generated-assets/` に保存されます。

### 保存ルール

リクエストID単位で次の2ファイルが作成されます。

- `.agents/generated-assets/<request_id>.<ext>`: 画像ファイル
- `.agents/generated-assets/<request_id>.md`: メタデータ (プロンプト/画像リンク)

### 参照画像の活用

画像生成時はテキストプロンプトだけでなく、**画像も添付**して依頼できます。さらに**特定の画像を参照として指定**でき、スタイルや構図、キャラクターの**一貫性を保ちながら編集や反復**が可能です。  
参照画像を指定しない場合は、選択済みの画像スコープや既定の入力が自動的に参照されることがあります。

### 実務のヒント

- 生成画像とメタデータをリポジトリに含めるかはチーム方針で決めてください。
- `.md`にプロンプトが記録されるため、機密情報は避けましょう。
- <BrandName /> に指示して、生成物を**プロジェクト内の特定フォルダへ移動**して整理できます。
- **ドキュメントベースのプロンプト管理やリソース編集**が可能になり、世界観/キャラクターガイドとプロンプトを一緒に管理してコンテンツやサービス素材の制作に活用できます。

## 既存のAPIキーとの関係

<BrandName />プロバイダーがリリースされても、既存の直接APIキー入力方式は引き続きサポートされます。次のような選択肢を持つことができます：
