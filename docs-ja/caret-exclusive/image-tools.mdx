# ^ イメージツール

CaretのAIは**画像を生成**し**分析**することができます。

## Clineとの違い

| 機能 | Cline | Caret |
|------|-------|-------|
| 画像生成 | 未サポート | **generate_imageツールでサポート** |
| 画像分析 (ビジョン非対応モデル) | 未サポート | **analyze_imageツールでサポート** |
| 比率/サイズ設定 | 未サポート | **UIで設定可能** |
| 参照画像ベースの生成 | 未サポート | **Image-to-Imageサポート** |

## 画像生成

### 使用例

```
ユーザー: かわいい猫の画像を作って
AI: [generate_imageツール使用] → 画像生成 → assets/に保存
```

### 設定オプション

| オプション | 値 | 説明 |
|------|-----|------|
| **比率** | `16:9`, `9:16`, `4:3`, `3:4`, `1:1` | 画像のアスペクト比 |
| **サイズ** | `1K`, `2K`, `3K`, `4K` | 画像の解像度 |

設定場所: Settings > Image Generation

### 保存場所

生成された画像はプロジェクトの`assets/`フォルダに保存されます:

```
assets/
├── img_abc123.png        # 生成された画像
└── img_abc123.md         # メタデータ (プロンプト、設定など)
```

## 画像分析

### ビジョンモデル vs テキストモデル

| モデルタイプ | 画像分析方法 |
|----------|----------------|
| **ビジョンモデル** (GPT-4o, Claude 3.5など) | `read_file`で直接分析 |
| **テキストモデル** (o1, GLM-4など) | `analyze_image`ツールを使用 |

### 使用例

```
# ビジョン非対応モデル (GLM-4.7など)
ユーザー: [画像添付] この画像に何がある？
AI: [analyze_imageツール使用] → Gemini Flashで分析 → 結果を返却
```

### 分析モデルの選択

テキストモデル使用時、画像分析用のモデルを選択できます:

- `gemini-2.5-flash` (デフォルト)
- `gemini-3.0-flash-preview`

設定場所: Settings > Model Info > Image Analysis Model

## 認証要件

| ツール | 要件 |
|------|----------|
| `generate_image` | Caretログインが必要 |
| `analyze_image` | Caretログインが必要 (テキストモデルのみ) |

## 設定

### 自動承認設定

| 設定 | デフォルト | 説明 |
|------|--------|------|
| `generateImages` | `true` | 画像生成ツールを有効化 |
| `analyzeImages` | `true` | 画像分析ツールを有効化 |

設定場所: Settings > Auto-approve

## セキュリティ

### 画像分析の承認

| ファイル位置 | 動作 |
|-----------|------|
| ワークスペース内 | ✅ 自動承認 |
| ワークスペース外 | ⚠️ ユーザー承認が必要 |

### 画像形式

サポートされる画像拡張子:
- `.png`, `.jpg`, `.jpeg`, `.webp`
- `.gif`, `.avif`, `.bmp`, `.tiff`

## 既知の制限事項

1. **認証必須**: 画像ツールはCaretログインが必要です
2. **単一実行**: 画像ツールは一度に1つだけ実行可能です
3. **サイズ制限**: 画像の最大ピクセルは7500px
