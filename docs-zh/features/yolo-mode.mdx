# YOLO 模式

YOLO 模式如其名——Careti 自动批准所有操作。在功能设置中选中复选框后，它将不经询问地执行文件更改、终端命令，甚至从 Agent 切换到 Act 模式。

可以把它看作[自动批准](/zh/features/auto-approve)功能的强化版——YOLO 模式不再提供细粒度的权限控制，而是赋予 Careti 完全的自主权。

<Note>
**警告：这很危险。** YOLO 模式会禁用所有安全保护措施。Careti 将毫无许可地执行它决定的所有操作。
</Note>

## 自动批准的内容

启用 YOLO 模式后，Careti 将自动批准：

- **所有文件操作** - 系统任何位置的文件读取、写入、修改
- **所有终端命令** - 包括潜在的破坏性操作
- **浏览器操作** - 网页抓取、表单提交、导航
- **MCP 服务器工具** - 外部集成和 API 调用
- **模式切换** - 从 Agent 自动切换到 Act 模式

本质上，所有安全护栏都被移除了。Careti 将以完全自主的方式运行。

## 如何启用 YOLO 模式

进入 Careti 设置 → 功能，勾选"YOLO 模式"复选框。就这么简单——没有确认对话框，也没有额外的警告。启用后，Careti 将立即开始自动批准所有操作。

要禁用它，只需取消勾选复选框即可。YOLO 模式关闭后，待处理的操作仍然需要批准。

## 何时使用？

YOLO 模式主要是为需要完全自主执行的可脚本化 CLI 而设计的。在 GUI 中，您可以在以下情况下使用它：

**快速原型开发**：当您完全不介意摩擦和错误时。非常适合一次性实验或快速探索新想法。

**经过验证的重复任务**：当您已经验证了 Careti 的方法，并希望消除批准开销时。想想日常重构或已建立的模式。

**演示目的**：当您想展示 Careti 的功能而不想被持续中断时。

## 可能出什么问题？

由于 YOLO 模式移除了所有安全保护措施，Careti 可能会：

- 在没有警告的情况下删除重要文件
- 执行修改系统设置的命令
- 向外部服务发送网络请求
- 覆盖配置文件
- 安装或卸载软件包
- 提交更改并推送到版本控制

风险级别完全取决于您要求 Careti 执行的任务。简单的任务相对安全，但复杂的请求可能会导致不可预测的结果。

## 最佳实践

如果您决定使用 YOLO 模式：

**从隔离环境开始。** 首先在一次性项目或沙盒环境中使用它。在理解风险之前，绝不要在生产代码库中启用它。

**提出具体的请求。** 模糊的指示加上无限制的权限可能导致意外结果。您的需求越清晰，Careti 的行为就越可预测。

**监控输出。** 即使 Careti 不要求许可，它仍然会显示正在做什么。实时观察终端输出和文件更改。

**准备好版本控制。** 确保在出错时可以轻松回滚更改。当 YOLO 模式是您的工作流程时，Git 就是您的安全网。

## 启发：什么成为可能？

启用 YOLO 模式后：

**用单个提示构建完整应用程序**：描述您想要什么，Careti 处理一切——创建文件、安装依赖项、配置设置，甚至部署脚本。

**自动化复杂工作流程**：自动化通常需要数十次批准点击的任务。数据处理管道、构建系统设置或多步骤重构变得流畅。

**快速迭代周期**：在没有批准摩擦的情况下快速测试想法。非常适合探索不同的方法或尝试新技术。

**现场演示**：展示 Careti 的全部功能，而无需暂停批准每个操作。非常适合演示或培训场景。

关键是要理解 YOLO 模式将 Careti 从交互式助手转变为自主代理。明智地使用这种力量。
