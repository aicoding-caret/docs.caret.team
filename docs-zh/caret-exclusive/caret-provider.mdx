# Careti 提供商

Careti 提供商是一个集成认证系统,让您无需 API 密钥即可立即使用 AI 编码助手。现在已可用。

## 这是什么?

许多首次使用 Careti 的开发者遇到的第一个障碍就是 API 密钥设置。获取 OpenAI 或 Anthropic 等 AI 提供商的 API 密钥、注册支付信息以及管理使用量的过程可能会很繁琐。

Careti 提供商消除了这个初始进入门槛。只需一个 Careti 账户,您就可以立即使用 AI 助手,无需单独设置 API 密钥。

## 优先支持 Google Gemini

Careti 提供商首个支持的模型是 **Google Gemini**。Gemini 具有强大的编码能力,同时价格相对实惠,非常适合初学者和个人开发者。

## 免费积分

所有首次使用 Careti 提供商的用户都会获得**免费积分**。这样您就可以在没有实际成本负担的情况下充分测试 Careti 的功能。

## 图像生成工具 (Careti 账号)

使用 Careti 账号登录后可以使用图像生成工具。该功能通过 Careti Token 认证,生成结果会自动保存到工作区。

### 如何工作

1. 让 Careti 生成图像 (例如: \"为仪表盘制作插画\").
2. Careti 调用图像生成 API 并返回结果。
3. 输出文件保存到工作区的 `.agents/generated-assets/`。

### 保存规则

每次请求会创建两个文件,以请求 ID 命名:

- `.agents/generated-assets/<request_id>.<ext>`: 图像文件
- `.agents/generated-assets/<request_id>.md`: 元数据(提示词/图像链接)

### 参考图像

生成图像时不仅可以提供文本提示,也可以附加图像。还可以**指定特定图像作为参考**,以保持风格、构图或角色的一致性,便于迭代或编辑。  
如果不指定参考图像,将根据已选择的图像范围或默认输入进行自动引用。

### 实用建议

- 是否将生成的图像和元数据提交到仓库,请按团队规范决定。
- 提示词会记录在 `.md` 中,请避免包含敏感信息。
- 可以让 Careti 将生成结果**移动到项目内指定文件夹**进行整理。
- 这使得**基于文档的提示词管理与资源编辑**成为可能,可结合世界观/角色指南管理提示词,用于内容制作或服务资源建设。

## 与现有 API 密钥的关系

即使 Careti 提供商推出后,现有的直接 API 密钥输入方式仍将继续支持。您将可以选择:

**使用 Careti 提供商**: 简单开始、统一管理、使用免费积分（Gemini 优先,ChatGPT·Claude·本地提供商如 Naver/Upstage 将陆续加入）

**直接使用 API 密钥**: 利用现有账户、更多模型选择、直接管理成本

您也可以同时使用这两种方式。
