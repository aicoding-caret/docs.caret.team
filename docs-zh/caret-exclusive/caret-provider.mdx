# Caret 提供商

Caret 提供商是一个集成认证系统,让您无需 API 密钥即可立即使用 AI 编码助手。目前正在开发中,即将推出。

## 这是什么?

许多首次使用 Caret 的开发者遇到的第一个障碍就是 API 密钥设置。获取 OpenAI 或 Anthropic 等 AI 提供商的 API 密钥、注册支付信息以及管理使用量的过程可能会很繁琐。

Caret 提供商消除了这个初始进入门槛。只需一个 Caret 账户,您就可以立即使用 AI 助手,无需单独设置 API 密钥。通过 Caret 账户,您将自动获得对 AI 模型的访问权限,并可以在统一的仪表板中管理使用量和成本。

## 优先支持 Google Gemini

Caret 提供商首个支持的模型将是 **Google Gemini**。Gemini 具有强大的编码能力,同时价格相对实惠,非常适合初学者和个人开发者。

Gemini 2.0 Flash 模型具有快速的响应速度和高效的成本结构,非常适合日常编码任务。对于需要复杂重构或大型代码库分析的情况,我们还计划提供 Gemini Pro 模型作为选项。

## 免费积分

我们将为所有首次使用 Caret 提供商的用户提供**免费积分**。这样您就可以在没有任何实际成本负担的情况下充分测试 Caret 的功能。

您可以在不注册信用卡的情况下获得免费积分,用完后可以以合理的价格购买额外积分。具体的免费积分金额和使用政策将在发布时公布。

## 未来计划

在 Gemini 支持之后,我们计划陆续添加其他 AI 模型,使其可通过 Caret 提供商使用。我们将提供对 Claude、GPT、DeepSeek 等各种模型的统一访问,这样用户就不需要单独管理多个提供商的 API 密钥。

我们还在与韩国主要 AI 模型提供商合作。我们的目标是让 Naver Clova、KT AI、Upstage Solar 等韩国公司的 AI 模型能够通过 Caret 提供商轻松使用。

## 与现有 API 密钥的关系

即使 Caret 提供商推出后,现有的直接 API 密钥输入方式仍将继续支持。您将可以选择:

**使用 Caret 提供商**: 简单开始、统一管理、使用免费积分（Gemini 优先,ChatGPT·Claude·本地提供商如 Naver/Upstage 将陆续加入）

**直接使用 API 密钥**: 利用现有账户、更多模型选择、直接管理成本

您也可以同时使用这两种方式。例如,日常任务使用 Caret 提供商的 Gemini 处理,当特别需要 Claude 时使用您自己的 Anthropic API 密钥。
