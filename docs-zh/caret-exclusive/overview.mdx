# Caret独有的特色功能

Caret不仅仅是简单地将Cline重新品牌化。我们在Cline坚实的基础上添加了实质性的扩展功能，为您提供更加出色的编码体验。

## 功能对比一览

| 功能 | Cline | Caret |
|------|-------|-------|
| 角色系统 | ❌ | ✅ |
| 双模式（Agent/Chatbot） | ❌ | ✅ |
| 提示历史记录（↑↓键） | ❌ | ✅ |
| 规则优先级系统 | ❌ | ✅ |
| 完整的多语言UI | ❌ | ✅（4种语言） |
| 自动提供商查询 | ❌ | ✅ |

## Caret独家功能详解

### 🎭 角色系统

自定义AI的名称和头像，打造属于您自己的编码伙伴。

- 提供5种默认角色（Caret、奥莎朗、真锅一香、希安·麦金、丹豆Ubuntu）
- 支持上传自定义图片（普通/思考中两种状态）
- 通过persona.md文件定义角色性格

[了解更多 →](/zh/caret-exclusive/persona-system)

### 🔄 双模式系统

根据不同的工作场景，在两种优化模式之间自由切换：

- **Agent模式**：自主的AI伙伴（灵活的对话转换）
- **Chatbot模式**：开发者主导的建议者（更好的AI控制）

*注意：Cline仅支持Plan/Act单一方式*

[了解更多 →](/zh/caret-exclusive/dual-prompt-modes)

### ⌨️ 提示历史记录系统

像在终端中一样，使用↑↓方向键快速重用之前的输入内容。

- 永久保存（即使重启VS Code也能保留）
- 最多保存1000条历史记录
- 自动去除重复项

### 📋 规则优先级系统

智能整合多个规则文件：

```
优先级：.agents/context
```

- 自动阻止重复规则（节省token）
- 仅将最高优先级规则包含在提示中

[了解更多 →](/zh/caret-exclusive/advanced-rules)

### 🤝 AI-开发者知识同步

Caret的AI如何与开发者共享相同知识以确保高可靠性？我们在此介绍作为Caret AI透明度与可信度基石的开源开发工作流。

[了解更多 →](/zh/caret-exclusive/knowledge-parity-system)

### 🌍 完整的多语言支持

真正的多语言支持，UI和文档都经过完整翻译：

- 支持韩语、英语、日语、中文
- 所有设置和消息都已翻译
- 针对每种语言提供专门的文档站点

[了解更多 →](/zh/caret-exclusive/multilingual-ui)

### 🎨 OEM品牌切换（企业版）

Caret和CodeCenter品牌可以实时切换的系统。

- 非公开定制项目
- 白标解决方案

**咨询联系**：support@caret.team

[了解更多 →](/zh/caret-exclusive/brand-switching)

### 🚀 提供商设置自动化（相关功能）

在选择AI模型时，逐一查找哪些模型可用是一件很麻烦的事情。Caret通过集成LiteLLM API实现了这一过程的自动化。

**自动模型查询**：当您选择一个提供商时，该提供商支持的所有AI模型都会自动显示出来。选择OpenAI后，GPT-4、GPT-3.5-Turbo等模型列表会立即出现；选择Anthropic后，则会显示Claude系列模型。

**实时更新**：由于通过LiteLLM API实时获取最新的模型信息，当新的AI模型发布时，无需更新Caret就会自动添加到列表中。

**广泛支持**：支持包括OpenAI、Anthropic、Google Gemini在内的**100多个AI提供商**。您可以轻松使用Cohere、AI21 Labs、Hugging Face等各种提供商的模型。

**与韩国AI企业合作**：Caret正在与韩国AI企业合作，支持更多的提供商。如果您想使用专门针对韩语优化的AI模型，未来将能够在Caret中轻松设置。

得益于这种自动化，开发者无需经过复杂的设置过程，就能快速选择所需的AI模型并开始编码。

[了解更多 →](/zh/caret-exclusive/enhanced-provider-setup)

## 为什么选择Caret？

### 即刻开始

使用免费的Gemini额度，无需设置API密钥即可立即开始编码（即将推出）。

### 与终端一致的用户体验

通过提示历史记录系统，提供与bash/zsh相同的↑↓键使用体验。

### 您的专属AI

通过角色系统，随心定制AI的名称和头像。

### 灵活的工作方式

Agent模式适合灵活对话，Chatbot模式适合开发者主导的工作。

### 母语支持

通过包括韩语在内的4种语言的完整支持，让您使用得更加舒适自如。
