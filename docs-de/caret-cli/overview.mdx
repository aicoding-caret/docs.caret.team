## Was ist Careti CLI?

Careti CLI führt KI-Codierungsagenten direkt in Ihrem Terminal aus. Leiten Sie `git diffs` für automatisierte Code-Reviews in CI/CD weiter, führen Sie mehrere Instanzen gleichzeitig für die parallele Entwicklung aus oder integrieren Sie Careti in Ihre bestehenden Shell-Workflows.

Die CLI verfolgt Instanzen in Ihrem gesamten System und gibt Ausgaben in Formaten aus, die sowohl für Menschen als auch für Skripte konzipiert sind – JSON, Klartext oder Rich-Terminal-Ausgabe.

<Tip>
Sind Sie bereit, loszulegen? Lesen Sie die [Installationsanleitung](/german/caret-cli/installation), um Careti CLI zu installieren und Ihre erste Aufgabe auszuführen.
</Tip>

## Unterstützte Modellanbieter

Careti CLI unterstützt mehrere KI-Modellanbieter und bietet Ihnen so Flexibilität bei der Auswahl des besten Modells für Ihre Bedürfnisse:

- **Anthropic**
- **OpenAI**
- **OpenAI Compatible**
- **OpenRouter**
- **X AI (Grok)**
- **AWS Bedrock**
- **Google Gemini**
- **Ollama**
- **Cerebras**

Während der Installation authentifizieren und konfigurieren Sie Ihren bevorzugten Anbieter mit dem Befehl `caret auth`.

## Was Sie damit erstellen können

**Automatisierte Code-Wartung**
- Planen Sie tägliche Ausführungen, um Linting-Probleme in Ihrer gesamten Codebasis zu identifizieren und zu beheben
- Erstellen Sie Aufgaben, die nach Sicherheitslücken suchen und diese automatisch beheben
- Erstellen Sie Skripte, die veraltete Abhängigkeiten aktualisieren und Tests ausführen

**Multi-Instanz-Entwicklung**
- Führen Sie separate Careti-Instanzen für Frontend und Backend gleichzeitig aus
- Starten Sie Instanzen für verschiedene Feature Branches, jede mit isoliertem Zustand
- Erstellen Sie parallele Review-Prozesse für mehrere PRs

**Benutzerdefinierte Workflows**
- Erstellen Sie Shell-Skripte, die Careti mit `git hooks` für die Pre-Commit-Analyse kombinieren
- Erstellen Sie benutzerdefinierte Befehle, die komplexe Datenstrukturen zur Verarbeitung durch Careti leiten
- Integrieren Sie Ihre bestehende Toolchain (`jq`, `grep`, `awk`) für anspruchsvolle Automatisierung

**CI/CD-Integration**
- Fügen Sie Careti zu GitHub Actions für die automatische Code-Überprüfung bei jedem PR hinzu
- Erstellen Sie GitLab-Pipelines, die Migrationsskripte aus Schemaänderungen generieren
- Erstellen Sie Jenkins-Jobs, die Careti verwenden, um Testfehler zu analysieren und Korrekturen vorzuschlagen
