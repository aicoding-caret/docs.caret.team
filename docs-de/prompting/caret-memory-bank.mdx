# Careti Context System (Memory Bank Ersatz)

Careti **ersetzt** den Cline Memory Bank Ansatz vollständig.  
Stattdessen verwaltet Careti den Projektkontext über **`AGENTS.md` + `.agents/context` + `caret-docs` + `work-logs`**.

## Schnellstart

1. **Führen Sie `/init` aus, um das Gerüst zu erstellen**  
   Dies generiert die grundlegende `AGENTS.md` und die `.agents/context` Struktur.
2. **Lesen Sie die obligatorische Dokumentation vor der Arbeit**  
   Prüfen Sie `AGENTS.md` → `.agents/context/caret-rules.json`.
3. **Laden Sie Workflows bei Bedarf**  
   Laden Sie nur die spezifischen Workflows aus `.agents/context/workflows/`, die Sie tatsächlich benötigen.
4. **Halten Sie Dokumentation und Logs aktuell**  
   Erfassen Sie Änderungen in `caret-docs/` und `work-logs/`.

## Kernkomponenten

| Komponente | Rolle | Anmerkungen |
| --- | --- | --- |
| `AGENTS.md` | Entry/Meta-Regeln | Ladereihenfolge, Sprache, Verbote |
| `.agents/context` | AI Rules SoT | Workflows + Atoms |
| `.agents/context/workflows` | On-demand Workflows | Nur bei Bedarf laden |
| `caret-docs/` | Entwicklerdokumentation | 1:1 Parität mit AI Rules |
| `work-logs/` | Fortschritt & Entscheidungen | Submodule, `.work-logs/` geplant |

## Warum Careti die Memory Bank ersetzt (Careti Vorteile)

- **AAIF internationaler Standard**: Pfade für Regeln und Ladevorgänge sind standardisiert.  
- **On-demand Laden von Regeln**: Es werden nur die notwendigen Dokumente gelesen.  
- **AI/Entwickler Wissensparität**: Beide sehen exakt dieselben Dokumente.  
- **Getrennte Fortschritts-Logs**: Status und Entscheidungen befinden sich in `work-logs/`.

## Legacy Memory Bank Mapping

| Memory Bank | Careti Ersatz |
| --- | --- |
| `projectbrief.md` | `caret-docs/development/index.md` + Übersichts-Spezifikationen |
| `productContext.md` | `caret-docs/features.en/**` |
| `systemPatterns.md` | `.agents/context/workflows/atoms/**` + Architektur-Dokumentation |
| `techContext.md` | `caret-docs/development/**` |
| `activeContext.md`/`progress.md` | `work-logs/**` (Master/Sub-Logs) |

> **Wichtig**: Verwenden Sie keinen `memory-bank/` Ordner. Migrieren Sie bestehende Dateien in die oben genannten Pfade.

## Arbeitsablauf

### Plan-Modus
1. Lesen Sie `AGENTS.md` → `.agents/context/caret-rules.json`.
2. Laden Sie nur die Workflows, die Sie benötigen.
3. Prüfen Sie `caret-docs/` für Spezifikationen und Anleitungen.
4. Teilen Sie ein Plan-Dokument vor der Ausführung mit.

### Act-Modus
1. Laden Sie nur die erforderlichen Regeln/Workflows.
2. Implementieren Sie die Änderungen.
3. Erfassen Sie Updates in `caret-docs/` und `work-logs/`.

## Update-Regeln

- **Regel-/Prozessänderungen** → `.agents/context` + `caret-docs/` gemeinsam aktualisieren
- **Feature-/Anforderungsänderungen** → `caret-docs/features.en/**` aktualisieren
- **Fortschritt/Entscheidungen** → in `work-logs/**` festhalten
- **Sicherheit** → Speichern Sie niemals Secrets/Token in der Dokumentation
