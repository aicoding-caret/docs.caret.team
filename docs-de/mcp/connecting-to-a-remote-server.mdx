## Remote MCP Server hinzufügen und verwalten

### Zugriff auf das MCP Servers Interface

Um auf das MCP Servers Interface in Careti zuzugreifen:

1. Klicken Sie auf das Careti-Icon in der VS Code Sidebar
2. Öffnen Sie das Menü (⋮) in der oberen rechten Ecke des Careti-Panels
3. Wählen Sie "MCP Servers" aus dem Dropdown-Menü

### Das MCP Servers Interface verstehen

Das MCP Servers Interface ist in drei Haupt-Tabs unterteilt:

-   **Marketplace**: Entdecken und installieren Sie vorkonfigurierte MCP Server (falls aktiviert)
-   **Remote Servers**: Verbinden Sie sich mit bestehenden MCP Servern über URL-Endpunkte
-   **Installed**: Verwalten Sie Ihre verbundenen MCP Server

### Einen Remote MCP Server hinzufügen

Der Tab "Remote Servers" ermöglicht es Ihnen, sich mit jedem MCP Server zu verbinden, der über einen URL-Endpunkt erreichbar ist:

1. Klicken Sie im MCP Servers Interface auf den Tab "Remote Servers"
2. Füllen Sie die erforderlichen Informationen aus:
    - **Server Name**: Geben Sie einen eindeutigen, beschreibenden Namen für den Server an
    - **Server URL**: Geben Sie den vollständigen URL-Endpunkt des MCP Servers ein (z. B. `https://example.com/mcp-sse`)
    - **Transport Type**: Wählen Sie das Verbindungsprotokoll aus (Streamable HTTP wird für moderne Server empfohlen)
3. Klicken Sie auf "Add Server", um die Verbindung zu initiieren
4. Careti wird versuchen, eine Verbindung zum Server herzustellen und den Verbindungsstatus anzeigen

> **Hinweis**: Wenn Sie eine Verbindung zu einem Remote Server herstellen, stellen Sie sicher, dass Sie der Quelle vertrauen, da MCP Server Code in Ihrer Umgebung ausführen können.

#### Transport-Typen

Careti unterstützt zwei Transportprotokolle für Remote MCP Server:

-   **Streamable HTTP (Empfohlen)**: Das moderne MCP-Transportprotokoll mit besserer Performance, Zuverlässigkeit und voller OAuth 2.1-Authentifizierungsunterstützung. Verwenden Sie dies für die meisten Remote Server.
-   **SSE (Legacy)**: Server-Sent Events Transport. Verwenden Sie dies nur, wenn der Server explizit SSE erfordert oder Streamable HTTP nicht unterstützt.

#### OAuth-Authentifizierung

Einige MCP Server (wie der von Vercel) erfordern eine OAuth-Authentifizierung, um sicher auf Ihre Daten zuzugreifen. Wenn Sie sich mit einem OAuth-fähigen Server verbinden:

1. Fügen Sie den Server wie gewohnt mit seiner URL hinzu
2. Wenn der Server eine Authentifizierung erfordert, wird eine Fehlermeldung angezeigt, die Sie zur Authentifizierung auffordert.
3. Klicken Sie auf die Schaltfläche **"Authenticate"**, die erscheint
4. Ihr Browser öffnet sich mit der Autorisierungsseite des Servers
5. Melden Sie sich an und erteilen Sie die Berechtigung
6. Sie werden automatisch zu Careti zurückgeleitet
7. Der Server verbindet sich und zeigt einen grünen Statuspunkt an

Sobald Sie authentifiziert sind, werden Ihre Zugangsdaten sicher gespeichert und der Server verbindet sich automatisch neu, wenn Sie Careti neu laden. Sie müssen sich nicht erneut authentifizieren, es sei denn, Sie löschen den Server oder Ihre Zugangsdaten laufen ab.

### Entdeckung von Remote Servern

Wenn Sie nach MCP Servern suchen, mit denen Sie sich verbinden können, bieten mehrere Marktplätze von Drittanbietern Verzeichnisse verfügbarer Server mit verschiedenen Funktionen an.

> **Warnung**: Die folgenden Marktplätze von Drittanbietern werden nur zu Informationszwecken aufgeführt. Careti unterstützt, verifiziert oder übernimmt keine Verantwortung für Server, die auf diesen Marktplätzen gelistet sind. Diese Server sind Cloud-gehostete Dienste, die Ihre Anfragen verarbeiten und Zugriff auf die von Ihnen geteilten Daten haben können. Prüfen Sie immer die Datenschutzrichtlinien und Nutzungsbedingungen, bevor Sie sich mit Diensten von Drittanbietern verbinden.

#### Composio MCP Integration

[Composio's MCP Marketplace](https://mcp.composio.dev/) bietet Zugriff auf eine Vielzahl von Drittanbieterservern, die das Model Context Protocol (MCP) unterstützen. Diese Server stellen APIs für Dienste wie GitHub, Notion, Slack und andere bereit. Jeder Server enthält Konfigurationsanweisungen und eine integrierte Authentifizierungsunterstützung (z. B. OAuth oder API-Keys). Um eine Verbindung herzustellen, suchen Sie den gewünschten Dienst im Marketplace und folgen Sie den dort beschriebenen Integrationsschritten.

#### Verbindung über Smithery

Smithery ist ein MCP Server Marktplatz eines Drittanbieters, der es Benutzern ermöglicht, verschiedene Model Context Protocol (MCP) Server zu entdecken und sich mit ihnen zu verbinden. Wenn Sie einen MCP-kompatiblen Client verwenden (wie Cursor, Claude Desktop oder Careti), können Sie die verfügbaren Server durchsuchen und sie direkt in Ihren Workflow integrieren.

Um die verfügbaren Optionen zu erkunden, besuchen Sie den Smithery Marketplace: [https://smithery.ai](https://smithery.ai)

Bitte beachten Sie: Smithery wird unabhängig gepflegt und steht in keiner Verbindung zu unserem Projekt. Die Nutzung erfolgt auf eigene Gefahr.

### Installierte MCP Server verwalten

Nach dem Hinzufügen erscheinen Ihre MCP Server im Tab "Configure", wo Sie folgende Möglichkeiten haben:

#### Serverstatus anzeigen

Jeder Server zeigt seinen aktuellen Status an:

-   **Grüner Punkt**: Verbunden und einsatzbereit
-   **Gelber Punkt**: Verbindung wird hergestellt
-   **Roter Punkt**: Getrennt oder es treten Fehler auf

#### Servereinstellungen konfigurieren

Klicken Sie auf einen Server, um dessen Einstellungsbereich zu erweitern:

1. **Tools & Resources**:
    - Alle verfügbaren Tools und Ressourcen des Servers anzeigen
    - Einstellungen für die automatische Genehmigung (Auto-Approval) von Tools konfigurieren (falls aktiviert)
2. **Request Timeout**:
    - Festlegen, wie lange Careti auf Serverantworten warten soll
    - Die Optionen reichen von 30 Sekunden bis zu 1 Stunde
3. **Server Management**:
    - **Restart Server**: Neu verbinden, wenn der Server nicht mehr reagiert
    - **Delete Server**: Entfernen Sie den Server aus Ihrer Konfiguration

#### Server aktivieren/deaktivieren

Betätigen Sie den Schalter neben jedem Server, um ihn zu aktivieren oder zu deaktivieren:

-   **Aktiviert**: Careti kann die Tools und Ressourcen des Servers verwenden
-   **Deaktiviert**: Der Server bleibt in Ihrer Konfiguration, ist aber nicht aktiv

### Fehlerbehebung bei Verbindungsproblemen

Wenn eine Verbindung zu einem Server fehlschlägt:

1. Eine Fehlermeldung mit Details zum Fehler wird angezeigt
2. **Bei OAuth-Fehlern**: Klicken Sie auf die Schaltfläche "Authenticate", um den Autorisierungsprozess abzuschließen
3. Überprüfen Sie, ob die Server URL korrekt ist und der Server läuft
4. Versuchen Sie, einen anderen Transport-Typ zu wählen (Streamable HTTP vs. SSE)
5. Verwenden Sie die Schaltfläche "Restart Server", um eine erneute Verbindung zu versuchen
6. Wenn die Probleme weiterhin bestehen, können Sie den Server löschen und versuchen, ihn erneut hinzuzufügen

#### OAuth-spezifische Probleme

Wenn Sie Probleme bei der Authentifizierung mit einem OAuth-fähigen Server haben:

-   **"Authentication required" bleibt bestehen**: Stellen Sie sicher, dass Sie den Autorisierungsprozess im Browser abgeschlossen und nicht abgebrochen haben
-   **Browser öffnet sich nicht**: Überprüfen Sie die Standard-Browser-Einstellungen Ihres Systems und stellen Sie sicher, dass externe URLs geöffnet werden können
-   **Redirect-Fehler**: Stellen Sie sicher, dass Sie die neueste Version von Careti verwenden – ältere Versionen unterstützen OAuth möglicherweise nicht
-   **Authentifizierung zurücksetzen**: Löschen Sie den Server und fügen Sie ihn erneut hinzu, um einen neuen OAuth-Prozess zu starten

### Erweiterte Konfiguration

Für fortgeschrittene Benutzer speichert Careti MCP Server-Konfigurationen in einer JSON-Datei, die geändert werden kann:

1. Klicken Sie im Tab "Configure" auf "Configure MCP Servers", um auf die Einstellungsdatei zuzugreifen
2. Die Konfiguration für jeden Server folgt diesem Format:

```json
{
	"mcpServers": {
		"exampleServer": {
			"url": "https://example.com/mcp-server",
			"type": "streamableHttp",
			"disabled": false,
			"autoApprove": ["tool1", "tool2"],
			"timeout": 60
		}
	}
}
```

Wichtige Konfigurationsoptionen:

-   **url**: Die Endpunkt-URL (für Remote Server)
-   **type**: Transportprotokoll - `"streamableHttp"` (empfohlen) oder `"sse"` (Legacy)
-   **disabled**: Ob der Server aktuell aktiviert ist (true/false)
-   **autoApprove**: Liste der Tool-Namen, die keine Bestätigung erfordern
-   **timeout**: Maximale Zeit in Sekunden, die auf Serverantworten gewartet wird (Standard: 60)

Für zusätzliche MCP-Einstellungen klicken Sie auf den Link "Advanced MCP Settings", um auf die VS Code Einstellungen zuzugreifen.

### MCP Server Tools verwenden

Sobald die Verbindung hergestellt ist, kann Careti die vom MCP Server bereitgestellten Tools und Ressourcen verwenden. Wenn Careti die Verwendung eines MCP Tools vorschlägt:

1. Eine Aufforderung zur Tool-Genehmigung erscheint (sofern nicht automatisch genehmigt)
2. Überprüfen Sie die Tool-Details und Parameter, bevor Sie sie genehmigen
3. Das Tool wird ausgeführt und gibt die Ergebnisse an Careti zurück

### Beispiel: Verbindung zu Vercel MCP herstellen

[Vercel MCP](https://vercel.com/docs/mcp/vercel-mcp) ist ein OAuth-fähiger Server, der Tools zur Verwaltung Ihrer Vercel-Projekte und Deployments bereitstellt:

1. Klicken Sie auf den Tab "Remote Servers"
2. Geben Sie Folgendes ein:
    - **Server Name**: `vercel`
    - **Server URL**: `https://mcp.vercel.com`
    - **Transport Type**: Streamable HTTP (vorausgewählt)
3. Klicken Sie auf "Add Server"
4. Sie sehen "Authentication required" – klicken Sie auf die Schaltfläche **"Authenticate"**
5. Melden Sie sich in Ihrem Browser bei Vercel an und autorisieren Sie Careti
6. Kehren Sie zu Careti zurück – der Server wird sich automatisch verbinden
7. Vercels Tools (deploy, logs, projects) stehen Careti nun zur Verfügung!

Ihre Vercel-Authentifizierung bleibt über Sitzungen hinweg bestehen, sodass Sie sich nicht jedes Mal neu authentifizieren müssen, wenn Sie Careti verwenden.
