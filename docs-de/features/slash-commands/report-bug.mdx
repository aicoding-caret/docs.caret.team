title: "Report Bug-Befehl"
sidebarTitle: "/reportbug"
---

`/reportbug` ist ein absoluter Lebensretter, wenn du auf ein merkwürdiges Problem mit Careti stößt. Anstatt dir alle Details merken zu müssen, die GitHub für einen Fehlerbericht benötigt, verwandelt dieser Befehl Careti in deinen persönlichen Assistenten für Fehlerberichte.

Er führt dich durch das Sammeln aller Informationen, die für einen ordnungsgemäßen Fehlerbericht benötigt werden, und sendet diese dann direkt an unsere GitHub Issues-Seite, wobei alle richtigen Formatierungen und Systemdetails enthalten sind.

#### Verwenden des `/reportbug` Slash Befehls

Wenn du auf etwas Seltsames stößt, das nicht richtig zu sein scheint:

-   Gib einfach `/reportbug` im Chat ein
-   Careti führt dich durch alle Details, die wir benötigen:
    -   Einen kurzen Titel, der das Problem beschreibt
    -   Was tatsächlich passiert ist im Vergleich zu dem, was du erwartet hast
    -   Schritte zur Reproduktion des Fehlers
    -   Alle relevanten Ausgaben oder Fehler, die du gesehen hast
    -   Zusätzlicher Kontext, der uns bei der Behebung helfen könnte
-   Du kannst alles überprüfen, bevor es eingereicht wird
-   Sobald du zustimmst, öffnet es ein perfekt formatiertes GitHub Issue mit all deinen Informationen plus automatischen Systemdetails

#### Beispiel

Letzte Woche bin ich auf einen seltsamen Bug gestoßen, bei dem Careti immer wieder ein Zeitlimit überschritt, wenn große Dateien gelesen wurden. Anstatt zu versuchen, mir alle GitHub-Template-Felder zu merken, habe ich einfach `/reportbug` eingegeben, und Careti hat mich durch den gesamten Prozess geführt.

Er fragte mich, was ich zu tun versuchte, was stattdessen geschah und welche Schritte genau zu dem Problem führten. Das Beste daran war, dass er automatisch meine OS-Version, Careti-Version und alle technischen Details einbezog, die unsere Entwickler benötigen würden.

Wenige Sekunden später hatte ich ein ordnungsgemäß formatiertes GitHub Issue erstellt, ohne diese Informationen selbst suchen zu müssen.
