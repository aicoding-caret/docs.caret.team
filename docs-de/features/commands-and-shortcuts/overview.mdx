### Was sind Editor-Integrationen?

Editor-Integrationen sind Befehle und Shortcuts, mit denen Sie Caret direkt dort verwenden können, wo Sie gerade arbeiten. Anstatt zuerst zum Caret-Panel zu wechseln, können Sie Code auswählen, mit der rechten Maustaste klicken und ihn sofort zur Unterstützung an Caret senden.
Diese Integrationen erscheinen an verschiedenen Stellen in VS Code:

-   Im Kontextmenü des Editors (Rechtsklickmenü) - "Add to Caret"
-   Im Kontextmenü des Terminals - "Add to Caret"
-   In der Source Control Ansicht - "Generate Commit Message"
-   Als Tastenkombinationen - Verschiedene Caret-Befehle
-   Als Quick Fix Optionen (Glühbirnenmenü) - "Fix with Caret", "Explain with Caret", "Improve with Caret"

### Wie sie funktionieren

Wenn Sie diese Befehle verwenden, führt Caret Folgendes aus:

-   Erfasst den relevanten Kontext (ausgewählter Code, Dateipfad, Terminalausgabe usw.)
-   Fokussiert die Caret-Oberfläche
-   Erstellt eine Konversation mit dem erfassten Kontext
-   In einigen Fällen wird automatisch ein Vorschlag generiert

Hinter den Kulissen verwenden diese Befehle die VS Code Extension API, um Befehle zu registrieren, auf den Editorstatus zuzugreifen und die VS Code-Oberfläche zu steuern.
