## Что такое Careti CLI?

Careti CLI запускает AI-агентов для кодирования непосредственно в вашем терминале. Передавайте `git diff` для автоматизированных проверок кода в CI/CD, запускайте несколько экземпляров одновременно для параллельной разработки или интегрируйте Careti в существующие рабочие процессы оболочки.

CLI отслеживает экземпляры в вашей системе и выводит данные в форматах, предназначенных как для людей, так и для скриптов — JSON, обычный текст или расширенный вывод в терминале.

<Tip>
Готовы начать? Ознакомьтесь с [руководством по установке](/russian/caret-cli/installation), чтобы установить Careti CLI и запустить свою первую задачу.
</Tip>

## Поддерживаемые провайдеры моделей

Careti CLI поддерживает несколько провайдеров AI-моделей, что дает вам гибкость в выборе лучшей модели для ваших нужд:

- **Anthropic**
- **OpenAI**
- **OpenAI Compatible**
- **OpenRouter**
- **X AI (Grok)**
- **AWS Bedrock**
- **Google Gemini**
- **Ollama**
- **Cerebras**

Во время установки вы пройдете аутентификацию и настроите предпочитаемого провайдера с помощью команды `caret auth`.

## Что вы можете создать с помощью этого

**Автоматизированное обслуживание кода**
- Запланируйте ежедневные запуски для выявления и исправления проблем линтинга во всей вашей кодовой базе
- Создавайте задачи, которые сканируют наличие уязвимостей безопасности и автоматически исправляют их
- Создавайте скрипты, которые обновляют устаревшие зависимости и запускают тесты

**Разработка с несколькими экземплярами**
- Запускайте отдельные экземпляры Careti для frontend и backend одновременно
- Создавайте экземпляры для разных веток feature, каждый с изолированным состоянием
- Создавайте параллельные процессы проверки для нескольких PR

**Пользовательские рабочие процессы**
- Создавайте shell-скрипты, которые объединяют Careti с `git hooks` для анализа перед коммитом
- Создавайте пользовательские команды, которые передают сложные структуры данных через Careti для обработки
- Интегрируйтесь с существующей цепочкой инструментов (jq, grep, awk) для сложной автоматизации

**Интеграция CI/CD**
- Добавьте Careti в GitHub Actions для автоматической проверки кода при каждом PR
- Создавайте конвейеры GitLab, которые генерируют скрипты миграции из изменений схемы
- Создавайте задания Jenkins, которые используют Careti для анализа сбоев тестов и предлагают исправления
