Упоминания файлов позволяют вам извлекать любой файл из вашего рабочего пространства непосредственно в ваш разговор с Caret. Больше не нужно копировать и вставлять фрагменты кода - просто введите `@/` и укажите файл, с которым вам нужна помощь.

Когда вы вводите `@/` в чате, Caret показывает файлы вашего рабочего пространства. Перемещайтесь по папкам, выберите нужный файл, и он мгновенно станет доступен Caret - со всеми импортами, связанными функциями и окружающим контекстом.

Я постоянно использую упоминания файлов при отладке. Вместо того чтобы пытаться понять, какие части моего кода нужно скопировать, я просто ссылаюсь на файл напрямую:

```
Я получаю эту ошибку при отправке моей формы: @terminal

Вот мой компонент: @/src/components/ContactForm.jsx

И API endpoint: @/src/api/contact.js

Что я упускаю?
```

Это дает Caret все необходимое - сообщение об ошибке, код компонента и API endpoint - и все это без необходимости что-либо копировать. Caret видит импорты, зависимости и весь окружающий контекст, который может вызывать проблему.

Упоминания файлов особенно полезны, когда вы имеете дело со сложными ошибками, охватывающими несколько файлов. Раньше мне приходилось тщательно копировать каждый соответствующий файл, следя за тем, чтобы ничего важного не пропустить. Теперь я просто ссылаюсь на каждый файл с помощью `@/`, и Caret получает полную картину.

В следующий раз, когда вы застрянете на проблеме, попробуйте использовать упоминания файлов вместо копирования кода. Вы сэкономите время и получите лучшие ответы, потому что у Caret будет весь необходимый контекст.

## Как Это Работает Под Капотом

Когда вы используете упоминание файла в своем сообщении, вот что происходит за кулисами:

1. Когда вы отправляете сообщение, Caret обнаруживает паттерн `@/path/to/file` в вашем тексте
2. Расширение разрешает путь к файлу относительно корня вашего рабочего пространства
3. Оно проверяет, является ли файл бинарным (например, изображением) или текстовым
4. Для текстовых файлов оно считывает полное содержимое файла
5. Содержимое файла добавляется к вашему сообщению в структурированном формате:
    ```
    <file_content path="path/to/file">
    [Complete file content]
    </file_content>
    ```
6. Это расширенное сообщение со встроенным содержимым файла отправляется AI
7. Теперь AI может "видеть" полное содержимое файла, как если бы вы скопировали и вставили его

Этот простой процесс происходит автоматически всякий раз, когда вы используете упоминание файла, предоставляя AI полный контекст без необходимости вручную что-либо копировать.
