Меню Auto Approve позволяет настраивать детализированные разрешения для действий, которые вы разрешаете Careti выполнять в автоматическом режиме.

## Как это работает

По умолчанию Careti будет запрашивать ваше разрешение перед вызовом любого инструмента, включая чтение или запись файлов.

Если вы хотите разрешить Careti выполнять какие-либо действия без подтверждения, вы можете установить разрешение Auto Approve для этого инструмента.

## Параметры разрешений

-   **Read project files**

    -   Позволяет Careti читать файлы в вашем текущем рабочем пространстве без запроса подтверждения
    -   **Read all files**
        -   Расширяет разрешение на чтение на файлы за пределами вашего рабочего пространства (системные файлы, конфигурационные файлы и т. д.)

-   **Edit project files**

    -   Позволяет Careti изменять файлы в вашем текущем рабочем пространстве без подтверждения
    -   **Edit all files**
        -   Расширяет разрешение на изменение на файлы за пределами вашего рабочего пространства

-   **Execute safe commands**

    -   Разрешает выполнение команд в терминале, которые модель считает неразрушающими
    -   **Execute all commands**
        -   Разрешает выполнение любой команды в терминале без запроса подтверждения

-   **Use the browser**

    -   Позволяет Careti использовать инструмент браузера для получения веб-контента

-   **Use MCP servers**

    -   Разрешает подключение и использование MCP servers для расширенной функциональности

-   **Maximum requests**
    -   Устанавливает количество последовательных автоматических действий, которые Careti может выполнить перед тем, как потребуется ваш ввод

## Рекомендации

Рекомендуемые начальные настройки для большинства рабочих процессов:

-   Автоматическое одобрение доступа на чтение (read access) к файлам проекта
-   Установка разумного лимита Maximum requests (10-20)

Это дает Careti возможность изучать проект без постоянных запросов, сохраняя при этом необходимость подтверждения для редактирования или деструктивных действий.

Настраивайте разрешения в зависимости от ситуации: делайте их строже для production и мягче для прототипирования. Используйте быстрые действия со «звездочкой», чтобы включать или выключать элементы Auto Approve по мере необходимости.
