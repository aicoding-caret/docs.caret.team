`/reportbug` – это просто спасение, когда вы сталкиваетесь со странной проблемой в Careti. Вместо того, чтобы вспоминать все детали, которые GitHub хочет для отчета об ошибке, эта команда превращает Careti в вашего личного помощника по созданию отчетов об ошибках.

Она поможет вам собрать всю информацию, необходимую для правильного отчета об ошибке, а затем отправит ее прямо на нашу страницу issues на GitHub со всем необходимым форматированием и деталями системы.

#### Использование Slash-команды `/reportbug`

Когда вы сталкиваетесь с чем-то странным, что кажется неправильным:

-   Просто введите `/reportbug` в чате
-   Careti проведет вас через все детали, которые нам нужны:
    -   Краткое название, описывающее проблему
    -   Что произошло на самом деле, в отличие от того, что вы ожидали
    -   Шаги для воспроизведения ошибки
    -   Любой релевантный вывод или ошибки, которые вы видели
    -   Дополнительный контекст, который может помочь нам исправить это
-   Вы сможете просмотреть все перед отправкой
-   После того, как вы одобрите, она откроет идеально отформатированный issue на GitHub со всей вашей информацией, а также автоматическими деталями системы

#### Пример

На прошлой неделе я столкнулся со странной ошибкой, когда Careti постоянно выдавал тайм-аут при чтении больших файлов. Вместо того, чтобы пытаться вспомнить все поля шаблона GitHub, я просто набрал `/reportbug`, и Careti провел меня через весь процесс.

Он спросил меня о том, что я пытался сделать, что произошло вместо этого и каковы были точные шаги, которые привели к проблеме. Самое приятное было то, что он автоматически включил мою версию ОС, версию Careti и все технические детали, которые понадобятся нашим разработчикам.

Через несколько секунд у меня был правильно отформатированный issue на GitHub, созданный без необходимости самому искать какую-либо из этих данных.
