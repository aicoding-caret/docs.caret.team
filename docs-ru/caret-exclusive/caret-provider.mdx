# Caret Provider

Caret Provider — это интегрированная система аутентификации, которая позволяет сразу начать работу с AI-помощником для написания кода без использования API key. Она доступна уже сейчас.

## Что это такое?

Первый барьер, с которым сталкиваются многие разработчики при первом использовании Caret, — это настройка API key. Процесс получения API keys от провайдеров AI, таких как OpenAI или Anthropic, регистрации платежной информации и управления использованием может быть обременительным.

Caret Provider устраняет этот порог входа. Имея всего лишь аккаунт Caret, вы можете немедленно приступить к использованию AI-помощника без отдельной настройки API key.

## Приоритетная поддержка Google Gemini

Первой поддерживаемой моделью в Caret Provider стала **Google Gemini**. Gemini предлагает мощные возможности для написания кода и при этом относительно доступна, что делает её подходящей для новичков и индивидуальных разработчиков.

## Бесплатные кредиты

Все пользователи, впервые использующие Caret Provider, получают **бесплатные кредиты**. Это позволяет протестировать функции Caret без реальных затрат.

## Инструмент генерации изображений (Caret Account)

При входе через аккаунт Caret вы можете использовать инструмент генерации изображений. Аутентификация происходит с помощью Caret token, а результаты автоматически сохраняются в вашем рабочем пространстве.

### Как это работает

1. Попросите Caret сгенерировать изображение (например, «Create an illustration for a dashboard»).
2. Caret вызывает API генерации изображений и выдает результат.
3. Файлы результатов сохраняются в папке `.agents/generated-assets/` вашего рабочего пространства.

### Правила хранения

Для каждого запроса создается два файла с использованием request ID:

- `.agents/generated-assets/<request_id>.<ext>`: файл изображения
- `.agents/generated-assets/<request_id>.md`: метаданные (prompt и ссылка на изображение)

### Референсные изображения

Помимо текстовых prompts, вы можете прикреплять изображения при запросе генерации. Вы можете указать **конкретное изображение в качестве референса (reference)**, чтобы сохранить **единый стиль, композицию или идентичность персонажа** в процессе итераций или редактирования изображений.  
Если вы не укажете референсное изображение, инструмент может автоматически использовать выбранную область изображения или другое стандартное поведение ввода.

### Практические советы

- Решите всей командой, следует ли фиксировать (commit) сгенерированные изображения и метаданные в репозиторий.
- Prompts сохраняются в файле `.md`, поэтому избегайте передачи конфиденциальной информации.
- Вы можете поручить Caret переместить сгенерированные ассеты в **определенные папки проекта** для более упорядоченного управления.
- Это позволяет реализовать **управление промптами на основе документов и редактирование ресурсов**, давая возможность вести руководства по созданию мира или персонажей вместе с промптами для создания контента или сервисных ресурсов.

## Взаимодействие с существующими API Keys

Даже после выхода Caret Provider поддержка существующего метода прямого ввода API key сохранится. У вас будут следующие варианты:

**Использование Caret Provider**: легкий старт, интегрированное управление, использование бесплатных кредитов (сначала Gemini, затем ChatGPT, Claude и локальные провайдеры, такие как Naver/Upstage, будут добавлены последовательно).

**Использование прямых API Keys**: использование существующих аккаунтов, больший выбор моделей, прямое управление расходами.

Также возможно использовать оба метода одновременно.
