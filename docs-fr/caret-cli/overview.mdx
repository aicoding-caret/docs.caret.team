## Qu'est-ce que Careti CLI ?

Careti CLI exécute des agents de codage IA directement dans votre terminal. Envoyez des git diffs pour des revues de code automatisées dans CI/CD, exécutez plusieurs instances simultanément pour un développement parallèle, ou intégrez Careti dans vos flux de travail shell existants.

La CLI suit les instances à travers votre système et les affiche dans des formats conçus à la fois pour les humains et les scripts : JSON, texte brut ou sortie terminal riche.

<Tip>
Prêt à démarrer ? Consultez le [guide d'installation](/french/caret-cli/installation) pour installer Careti CLI et exécuter votre première tâche.
</Tip>

## Fournisseurs de modèles pris en charge

Careti CLI prend en charge plusieurs fournisseurs de modèles d'IA, vous offrant ainsi une flexibilité dans le choix du meilleur modèle pour vos besoins :

- **Anthropic**
- **OpenAI**
- **OpenAI Compatible**
- **OpenRouter**
- **X AI (Grok)**
- **AWS Bedrock**
- **Google Gemini**
- **Ollama**
- **Cerebras**

Lors de l'installation, vous vous authentifierez et configurerez votre fournisseur préféré à l'aide de la commande `caret auth`.

## Ce que vous pouvez construire avec ceci

**Maintenance de code automatisée**
- Planifiez des exécutions quotidiennes pour identifier et corriger les problèmes de linting dans votre codebase.
- Créez des tâches qui recherchent les vulnérabilités de sécurité et les corrigent automatiquement.
- Créez des scripts qui mettent à jour les dépendances obsolètes et exécutent des tests.

**Développement multi-instance**
- Exécutez des instances Careti distinctes pour le frontend et le backend simultanément.
- Créez des instances pour différentes branches de fonctionnalités, chacune avec un état isolé.
- Créez des processus d'examen parallèles pour plusieurs PRs.

**Flux de travail personnalisés**
- Créez des scripts shell qui combinent Careti avec des git hooks pour une analyse pre-commit.
- Créez des commandes personnalisées qui acheminent des structures de données complexes via Careti pour le traitement.
- Intégrez-vous à votre toolchain existant (jq, grep, awk) pour une automatisation sophistiquée.

**Intégration CI/CD**
- Ajoutez Careti à GitHub Actions pour une revue de code automatique sur chaque PR.
- Créez des pipelines GitLab qui génèrent des scripts de migration à partir des modifications de schéma.
- Créez des jobs Jenkins qui utilisent Careti pour analyser les échecs de tests et suggérer des corrections.
