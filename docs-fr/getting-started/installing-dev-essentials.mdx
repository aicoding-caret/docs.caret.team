### Outils Essentiels

Voici les outils de base dont vous avez besoin pour le développement.

-   **Node.js & npm :** Requis pour JavaScript et le développement web.
-   **Git :** Utilisé pour suivre les modifications du code et collaborer avec d'autres.
-   **Gestionnaire de paquets :** Permet d'installer facilement d'autres outils de développement.
    -   Homebrew pour macOS
    -   Chocolatey pour Windows
    -   apt/yum pour Linux

> **Astuce :** Ces outils constituent la base de votre boîte à outils de développeur. Les installer correctement vous prépare au succès.

### Que va-t-il se passer ?

Cline vous guidera à travers ces étapes :

1.  Installer le bon gestionnaire de paquets pour votre système
2.  Utiliser le gestionnaire de paquets pour installer Node.js et Git
3.  Afficher les commandes exactes avant de les exécuter (vous approuvez chaque étape)
4.  Vérifier que chaque installation a réussi

> **Note :** Certaines étapes peuvent nécessiter le mot de passe de votre ordinateur. Ceci est normal.

### Pourquoi ces outils sont importants

-   **Node.js & npm :**
    -   Créer des sites web avec des frameworks comme React ou Next.js
    -   Exécuter du code JavaScript
    -   Installer des paquets JavaScript
-   **Git :**
    -   Enregistrer différentes versions de votre code
    -   Collaborer avec d'autres développeurs
    -   Sauvegarder votre travail
-   **Gestionnaire de paquets :**
    -   Installer et mettre à jour rapidement les outils de développement
    -   Maintenir votre environnement propre et à jour

### Conseils supplémentaires pour les nouveaux codeurs

#### Comprendre le Terminal

Le terminal est une application où vous tapez des commandes pour interagir avec votre ordinateur.

-   **macOS :** Ouvrez-le en recherchant "Terminal" dans Spotlight.
-   **Exemple :**

```
$ open -a Terminal
```

#### Comprendre les fonctionnalités de VS Code

-   **Terminal VS Code :** Exécutez des commandes directement à l'intérieur de VS Code.
    -   Allez dans **View > Terminal** ou appuyez sur ``Ctrl + ` ``.
    -   **Exemple :**

```
$ node -v
v16.14.0
```

-   **Vue Éditeur :** Où vous modifiez les fichiers de code.
    -   Ouvrez les fichiers à partir du panneau Explorateur sur la gauche.
-   **Section Problèmes :** Affichez les erreurs ou les avertissements dans votre code.
    -   Cliquez sur l'icône en forme d'ampoule ou allez dans **View > Problems**.

#### Termes courants

-   **Interface de ligne de commande (CLI) :** Un moyen puissant d'exécuter des commandes.
-   **Permissions :** Certaines commandes nécessitent des privilèges élevés. Cela aide à garder votre système en sécurité.
