`/reportbug` est une véritable bouée de sauvetage lorsque vous rencontrez un problème bizarre avec Careti. Au lieu de devoir vous souvenir de tous les détails que GitHub exige pour un rapport de bug, cette commande transforme Careti en votre assistant personnel de signalement de bugs.

Il vous guide dans la collecte de toutes les informations nécessaires à un rapport de bug correct, puis l'envoie directement à notre page GitHub issues avec toute la mise en forme appropriée et les détails du système inclus.

#### Utilisation de la commande Slash `/reportbug`

Lorsque vous rencontrez quelque chose d'étrange qui ne vous semble pas correct :

-   Tapez simplement `/reportbug` dans le chat
-   Careti vous guidera à travers tous les détails dont nous avons besoin :
    -   Un titre rapide décrivant le problème
    -   Ce qui s'est réellement passé par rapport à ce que vous attendiez
    -   Les étapes pour reproduire le bug
    -   Toute sortie ou erreur pertinente que vous avez vue
    -   Tout contexte supplémentaire qui pourrait nous aider à le corriger
-   Vous pourrez tout examiner avant de le soumettre
-   Une fois que vous avez approuvé, il ouvre un issue GitHub parfaitement formaté avec toutes vos informations ainsi que les détails automatiques du système

#### Exemple

La semaine dernière, j'ai rencontré un bug bizarre où Careti n'arrêtait pas d'expirer lors de la lecture de fichiers volumineux. Au lieu d'essayer de me souvenir de tous les champs du modèle GitHub, j'ai simplement tapé `/reportbug` et Careti m'a guidé tout au long du processus.

Il m'a demandé ce que j'essayais de faire, ce qui s'est passé à la place et les étapes exactes qui ont conduit au problème. Le meilleur, c'est qu'il incluait automatiquement ma version de système d'exploitation, ma version de Careti et tous les détails techniques dont nos développeurs auraient besoin.

Quelques secondes plus tard, j'avais un issue GitHub correctement formaté, créé sans avoir à rechercher moi-même toutes ces informations.
