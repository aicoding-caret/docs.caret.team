# Caret Provider

Caret Provider est un système d'authentification intégré qui vous permet d'utiliser un assistant de codage AI immédiatement sans clé API. Il est disponible dès maintenant.

## Qu'est-ce que c'est ?

Le premier obstacle auquel de nombreux développeurs sont confrontés lors de leur première utilisation de Caret est la configuration de la clé API. Le processus d'obtention de clés API auprès de fournisseurs d'AI tels qu'OpenAI ou Anthropic, l'enregistrement des informations de paiement et la gestion de l'utilisation peut être laborieux.

Caret Provider élimine cette barrière à l'entrée initiale. Avec un simple compte Caret, vous pouvez utiliser immédiatement l'assistant AI sans configuration de clé API séparée.

## Support prioritaire pour Google Gemini

Le premier modèle supporté par Caret Provider est **Google Gemini**. Gemini offre de puissantes capacités de codage tout en étant relativement abordable, ce qui le rend idéal pour les débutants et les développeurs individuels.

## Crédits gratuits

Tous les utilisateurs qui utilisent Caret Provider pour la première fois reçoivent des **crédits gratuits**. Cela vous permet de tester les fonctionnalités de Caret sans contrainte de coût réel.

## Outil de génération d'images (Compte Caret)

Lorsque vous vous connectez avec un compte Caret, vous pouvez utiliser l'outil de génération d'images. L'authentification se fait via le token Caret, et les fichiers de sortie sont automatiquement enregistrés dans votre workspace.

### Comment ça fonctionne

1. Demandez à Caret de générer une image (ex : "Crée une illustration pour un dashboard").
2. Caret appelle l'API de génération d'images et produit le résultat.
3. Les fichiers de sortie sont enregistrés sous `.agents/generated-assets/` dans votre workspace.

### Règles de stockage

Chaque requête crée deux fichiers utilisant l'ID de requête :

- `.agents/generated-assets/<request_id>.<ext>` : le fichier image
- `.agents/generated-assets/<request_id>.md` : métadonnées (prompt et lien de l'image)

### Images de référence

En plus des prompts textuels, vous pouvez joindre des images lors d'une demande de génération. Vous pouvez spécifier **une image particulière comme référence** pour conserver un **style, une composition ou une identité de personnage cohérente** lors de l'itération ou de l'édition d'images.  
Si vous ne spécifiez pas d'image de référence, l'outil peut utiliser automatiquement le périmètre d'image sélectionné ou un autre comportement d'entrée par défaut.

### Conseils pratiques

- Décidez en équipe si les images générées et les métadonnées doivent être commitées dans le repo.
- Les prompts sont stockés dans le fichier `.md`, évitez donc les informations sensibles.
- Vous pouvez demander à Caret de déplacer les assets générés dans des **dossiers de projet spécifiques** pour une gestion organisée.
- Cela permet une **gestion de prompts et une édition de ressources basées sur les documents**, vous permettant de gérer le worldbuilding ou les guides de personnages aux côtés des prompts pour produire du contenu ou des assets de service.

## Relation avec les clés API existantes

Même après la sortie de Caret Provider, la méthode existante de saisie directe des clés API continuera d'être supportée. Vous aurez les options suivantes :

**Utiliser Caret Provider** : démarrage facile, gestion intégrée, utilisation de crédits gratuits (Gemini d'abord, suivi de ChatGPT, Claude et de fournisseurs locaux comme Naver/Upstage ajoutés séquentiellement)

**Utiliser les clés API directes** : exploiter les comptes existants, plus de choix de modèles, gestion directe des coûts

Il est également possible d'utiliser les deux méthodes ensemble.
