이 명령은 **VS Code**에서만 사용할 수 있습니다. 인라인 코멘트가 포함된 diff 뷰는 VS Code의 기본 diff 기능이 필요합니다.

`/explain-changes`는 **어떤 git diff든 AI 설명을 생성**하는 슬래시 명령입니다. 완료 메시지의 [변경 사항 설명 버튼](/features/explain-changes)이 **완료된 작업**의 변경을 설명하는 반면, 이 명령은 **커밋/브랜치/태그/PR/스테이지/작업 디렉터리** 등 원하는 git 참조 간의 변경을 설명합니다.

## 요구사항

<Note>
`/explain-changes`는 **git 저장소**가 필요합니다. git이 초기화된 디렉터리에서 실행하세요.
</Note>

PR 설명에는 [GitHub CLI(gh)](https://cli.github.com/)가 설치·인증되어 있어야 합니다. GitLab MR 설명은 [GitLab CLI(glab)](https://gitlab.com/gitlab-org/cli)가 필요합니다.

이 명령은 git 참조를 직접 사용하므로 **체크포인트를 켤 필요가 없습니다.**

## 명령 사용 방법

채팅 입력창에 `/explain-changes`를 입력하면 Caret이 다음을 수행합니다:

1. git 기록을 분석해 변경 사항 파악
2. 관련 파일을 읽어 컨텍스트 확보
3. 비교할 git 참조 결정
4. 인라인 설명이 포함된 diff 뷰 생성

## 동작 방식

`/explain-changes` 실행 시 Caret은:

1. **컨텍스트 수집**: git 명령으로 저장소 상태 파악
2. **변경 식별**: 참조 간 변경 파일 식별
3. **관련 파일 읽기**: 더 나은 설명을 위해 컨텍스트 보강
4. **generate_explanation 호출**: 인라인 설명을 포함한 diff 생성
5. **결과 표시**: 멀티 파일 diff와 인라인 코멘트 표시

## 활용 사례

### 최근 커밋 설명

가장 흔한 사용 사례입니다:

```
/explain-changes
```

Caret은 HEAD와 HEAD~1을 비교해 최근 커밋의 변경을 설명합니다.

**사용 시점:**
- 팀원의 변경을 pull 받은 뒤
- push 전에 본인 변경을 검토할 때
- merge 커밋이 무엇을 가져왔는지 확인할 때

### 커밋되지 않은 변경 설명

작업 중 변경 내용을 커밋 전 점검할 때:

```
/explain-changes for my uncommitted work
```

Caret은 HEAD와 작업 디렉터리를 비교해 수정된 파일을 설명합니다.

**사용 시점:**
- `git add` 전 변경이 제대로 되었는지 확인
- 긴 작업 후 변경 내용 정리
- 커밋 전 마지막 검토

### 스테이지된 변경 설명

커밋 직전에 스테이지된 변경만 확인:

```
/explain-changes for my staged changes
```

**사용 시점:**
- 커밋 직전 최종 점검
- 일부만 스테이지했을 때 확인
- 의도치 않은 파일이 섞였는지 검증

### 특정 커밋 설명

```
/explain-changes for commit abc123
```

또는 메시지 기준:

```
/explain-changes for the commit that added authentication
```

**사용 시점:**
- 버그가 언제 들어왔는지 조사
- 과거 결정을 이해
- 기능 구현 방식 파악

### 커밋 범위 설명

```
/explain-changes for the last 3 commits
```

또는 범위 지정:

```
/explain-changes from v1.0.0 to v1.1.0
```

**사용 시점:**
- 릴리즈 간 변경 파악
- 여러 커밋 묶음 리뷰
- 장기간 부재 후 변경 요약

### Pull Request 설명

```
/explain-changes for PR #42
```

Caret이 GitHub CLI로 PR 정보를 가져와 설명합니다.

**사용 시점:**
- 다른 사람 PR 리뷰
- 승인 전 이해
- 오픈소스 PR 학습
- 피드백 준비

### 브랜치 비교

```
/explain-changes between main and feature-branch
```

또는

```
/explain-changes for everything on my-feature that's not in main
```

**사용 시점:**
- 머지 전 변경 파악
- 브랜치 간 차이 분석
- rebase/merge 전략 수립

### 특정 파일 변경 설명

```
/explain-changes for src/auth in the last 5 commits
```

Caret이 해당 파일/디렉터리만 필터링해 설명합니다.

**사용 시점:**
- 특정 모듈 변경 추적
- 핵심 파일 변동 이해
- 기능 진화 과정 분석

### 태그 이후 변경 설명

```
/explain-changes since v2.0.0
```

Caret이 태그와 HEAD를 비교해 이후 변경을 설명합니다.
