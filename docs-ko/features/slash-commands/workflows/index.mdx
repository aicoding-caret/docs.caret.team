Caret의 워크플로는 Markdown 파일로 작성되는 **작업 절차 스크립트**입니다. 반복적이거나 복잡한 작업을 자동화해, 에디터 안에서 일관된 프로세스를 실행할 수 있게 해줍니다.

워크플로를 실행하려면 채팅에 `/`와 워크플로 파일명을 입력하면 됩니다(예: `/deploy.md`).

## 왜 워크플로를 쓰나요?

* **자동화**: 새 프로젝트 설정, 배포, 테스트 실행 등 반복 작업을 자동화
* **일관성**: 매번 동일한 절차로 작업하여 실수 감소
* **인지 부담 감소**: 복잡한 단계/명령을 기억할 필요 없음
* **컨텍스트 기반**: 프로젝트 컨텍스트 안에서 실행되어 파일/도구 접근이 자연스러움

## 동작 방식

워크플로는 `.md` 확장자를 가진 표준 Markdown 파일입니다. Caret은 이 파일을 읽고 단계별로 실행합니다.

* **Caret 도구**: `read_file`, `write_to_file`, `execute_command`, `ask_followup_question` 등
* **CLI 도구**: `git`, `gh`, `npm`, `docker` 등 로컬 도구
* **MCP 도구**: 연결된 MCP 서버의 도구

## 워크플로 vs 규칙

워크플로는 “무엇을 할지”, 규칙은 “어떻게 행동할지”를 정의합니다.

| 구분 | 목적 | 언제 사용? |
| :--- | :--- | :--- |
| **Caret Rules** | Caret의 행동 원칙 정의 | 코딩 표준, 기술 스택, 프로젝트 제약 관리 |
| **Caret Workflows** | 특정 작업 절차 정의 | 반복 작업 자동화, 배포/리포트 생성 등 |

**Rules는 환경**, **Workflows는 스크립트**라고 생각하면 이해하기 쉽습니다.

### 예시: 릴리즈 자동화

워크플로 없이 수동으로 한다면:
1. `package.json` 버전 변경
2. 테스트 실행
3. `CHANGELOG.md` 업데이트
4. 커밋/태그/푸시

워크플로로는 `release.md` 파일을 만들고 아래처럼 실행하면 됩니다:

```bash
/release.md
```

Caret이 정의된 절차를 따라 자동으로 실행하며, 필요 시 중단하고 질문합니다.

## 워크플로 위치

워크플로는 프로젝트용/전역용 두 가지 위치에 둘 수 있습니다.

<Tabs>
  <Tab title="프로젝트 전용 워크플로">
    프로젝트 전용 워크플로는 **`.agents/context/workflows/`**에 저장합니다.

    1. 프로젝트 루트에 `.agents/context/workflows/` 폴더 생성
    2. 해당 폴더에 `deploy.md` 같은 워크플로 파일 추가

    이 워크플로는 해당 프로젝트에서만 사용됩니다.
  </Tab>
  <Tab title="전역 워크플로">
    여러 프로젝트에서 공유할 워크플로는 전역 폴더에 저장합니다.

    * **macOS/Linux:** `~/Documents/Caret/Workflows/`
    * **Windows:** `C:\Users\USERNAME\Documents\Caret\Workflows\`

    이 폴더에 `.md` 파일을 넣으면 모든 프로젝트에서 사용할 수 있습니다.
  </Tab>
</Tabs>

## 워크플로 관리

Caret 확장 안에서 워크플로를 직접 관리할 수 있습니다.

1. 확장 하단의 **Manage Caret Rules & Workflows** 버튼(<Icon icon="scale-balanced" />) 클릭
2. 다음을 수행 가능:
    * **전체 워크플로 목록 확인**
    * **워크플로 ON/OFF 토글**
    * **생성/편집**
    * **삭제**

## 워크플로 예시 구조

아래는 간단한 워크플로 예시(`daily-changelog.md`)입니다.

````markdown daily-changelog.md
# Daily Changelog Generator

This workflow helps you create a changelog for your daily work.

1.  **Check your recent git commits:**
    I will run the following command to see your commits from today.
    ```bash
    git log --author="$(git config user.name)" --since="yesterday" --oneline
    ```

2.  **Summarize your work:**
    I will present the commits to you and ask for a summary of your changes to be added to the `changelog.md` file.

3.  **Create/Append to daily changelog:**
    I will append to the `changelog.md` file. The content will include a header with the current date, the list of commits, and your summary.
````

### 워크플로 구조 설명

이 예시는 XML 도구 호출을 직접 적지 않아도, Caret이 문맥을 이해해 실행할 수 있음을 보여줍니다.

1. **Step 1: 최근 커밋 확인**
    * Caret이 명령을 실행해 정확한 데이터를 확보
    <Tip>
      git 로그를 보여준 뒤 **Proceed While Running** 버튼을 눌러 이어서 실행해야 할 수 있습니다.
    </Tip>

2. **Step 2: 요약 요청**
    * 특정 도구 호출 대신 "요약을 요청"이라고만 작성해도 충분

3. **Step 3: 로그 작성**
    * 최종 결과를 명확히 설명하면 Caret이 파일 작성 도구를 적절히 사용
