<Note>
이 기능은 **VS Code**에서만 사용할 수 있습니다. 인라인 코멘트가 포함된 diff 뷰는 VS Code의 기본 diff 기능이 필요합니다.
</Note>

**변경 사항 설명(Explain Changes)**은 코드 변경에 인라인 설명을 추가하는 AI 기반 코드 리뷰 기능입니다. Caret이 코드베이스를 수정한 뒤 버튼을 클릭하면, VS Code의 diff 뷰에 맥락 기반 설명이 스트리밍으로 표시됩니다.

## 동작 방식

<Note>
변경 사항 설명은 **[체크포인트](/features/checkpoints)** 기능이 활성화되어 있어야 합니다. 사용 전 Caret 설정에서 체크포인트를 켜세요.
</Note>

Caret이 파일 변경을 포함한 작업을 완료한 뒤 완료 메시지에 **"View Changes"** 옆에 **"Explain Changes"** 버튼이 나타납니다. 버튼을 클릭하면:

1. 변경된 모든 파일을 보여주는 멀티 파일 diff 뷰가 열리고
2. AI가 인라인 코멘트로 설명을 스트리밍하며
3. 변경 이유와 맥락이 필요한 위치에 코멘트를 배치합니다

AI는 단순히 코드가 무엇을 하는지 설명하는 데 그치지 않고, 변경 이유와 의도를 함께 설명합니다.

## 대화형 코멘트 스레드

변경 사항 설명의 핵심은 **코멘트가 완전히 대화형**이라는 점입니다. 각 코멘트 스레드에서 바로 대화를 이어갈 수 있습니다.

### 후속 질문하기

각 설명 코멘트에는 해당 코드에 대한 질문을 입력할 수 있는 답글 입력창이 있습니다:

- "왜 이 접근을 선택했나요?"
- "이 패턴을 더 자세히 설명해줘"
- "이걸 Y로 바꾸면 어떤 영향이 있나요?"

AI는 코드와 작업 컨텍스트를 동시에 이해한 답변을 제공합니다.

### 메인 채팅으로 이동

코멘트 스레드 대화가 복잡해지거나 추가 작업이 필요할 경우, 스레드의 제목 영역을 클릭하면 대화를 Caret의 메인 채팅으로 옮길 수 있습니다. 이렇게 하면:

- Caret의 전체 기능으로 대화를 계속할 수 있고
- 해당 논의를 바탕으로 추가 변경을 요청할 수 있으며
- 리뷰 대화의 컨텍스트를 그대로 유지할 수 있습니다

## 언제 사용하면 좋을까요?

### 학습과 온보딩

낯선 코드베이스나 새로운 패턴을 다룰 때, 단순한 변경 내용뿐 아니라 **왜 그렇게 변경했는지**를 이해하는 데 도움이 됩니다.

- 설계 결정과 트레이드오프
- 사용된 기술 개념과 패턴
- 변경 간 상호 관계

### 코드 리뷰

리뷰 과정에서 다음과 같이 활용할 수 있습니다:

- 커밋 전에 복잡한 변경을 이해
- AI의 추론이 기대와 일치하는지 확인
- 전체 맥락을 파악해潜在 이슈를 발견

### 지식 전달

설명 코멘트는 변경의 이유를 문서처럼 남깁니다. 팀원이 코드를 볼 때 각 수정의 의도를 쉽게 이해할 수 있습니다.

## 베스트 프랙티스

1. **구체적으로 질문하기**: 질문이 구체적일수록 AI 답변이 더 유용합니다.
2. **복잡한 변경에 활용**: 다수 파일 또는 복잡한 로직 변경에 특히 효과적입니다.
3. **중요한 논의는 채팅으로 이동**: 추가 작업이 필요하면 메인 채팅에서 이어가세요.
4. **커밋 전 최종 점검**: 커밋 전에 변경을 전부 이해했는지 확인하세요.
