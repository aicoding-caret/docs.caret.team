Caret은 작업 기록을 관리하기 위한 도구를 제공합니다. 시간이 지날수록 작업이 많아지므로, 검색/정렬/필터링 기능은 생산성에 매우 중요합니다.

## 작업 기록 열기

작업 기록에 접근하는 방법은 여러 가지가 있습니다:

1. **Caret 사이드바의 History 버튼** 클릭
2. **명령 팔레트에서 "History" 검색** (환경에 따라 표시되는 이름이 다를 수 있음)
3. **키보드 단축키** (사용자 설정이 있을 경우)

## 작업 기록 인터페이스

작업 기록 화면에서는 과거/현재 작업을 한눈에 관리할 수 있습니다. 검색, 정렬, 필터링을 통해 필요한 작업을 빠르게 찾을 수 있습니다.

### 검색과 필터

#### 검색 바
- 모든 작업 내용에 대해 **퍼지 검색** 수행
- 프롬프트, 응답, 코드 내용까지 포함
- 입력 즉시 결과 필터링
- 일치 텍스트 하이라이트

#### 정렬 옵션
- **최신순** (기본)
- **오래된순**
- **비용이 큰 작업**
- **토큰 사용량이 많은 작업**
- **가장 관련성 높은 작업**

#### 즐겨찾기 필터
- 별표 표시된 작업만 보기
- 중요한 작업에 빠르게 접근
- 검색과 함께 사용 가능

## 작업별 액션

각 작업에서 사용할 수 있는 주요 액션:

### 기본 액션

- **열기**: 작업을 다시 열어 대화 재개
- **재개**: 중단된 작업을 이어서 진행
- **내보내기**: 작업 대화를 Markdown으로 저장

### 관리 액션

- **즐겨찾기** ⭐: 별표 표시로 중요 작업 보호
- **삭제** 🗑️: 개별 작업 삭제 (즐겨찾기는 보호됨)
- **복제**: 기존 작업을 기반으로 새 작업 생성

## ⭐ 작업 즐겨찾기

즐겨찾기 기능은 중요한 작업을 보존하고 빠르게 접근하는 데 유용합니다.

### 즐겨찾기 사용 방법

**즐겨찾기 지정**
- 별 아이콘 클릭 → 즐겨찾기 설정
- 다시 클릭 → 해제

**보호 기능**
- 즐겨찾기 작업은 실수로 삭제되지 않도록 보호
- 일괄 삭제 시 즐겨찾기는 기본적으로 제외
- 명시적 확인으로 보호를 해제할 수 있음

**활용 예시**
- 참고용 구현 사례
- 성공적인 문제 해결 패턴
- 재사용 가능한 코드 스니펫
- 중요한 프로젝트 마일스톤
- 팀 학습용 예제

## 작업 메트릭

작업별 리소스 사용량을 확인할 수 있습니다.

### 제공되는 메트릭

- **토큰 사용량**: 입력/출력 토큰 합계
- **API 비용**: 모델 가격 기반 추정 비용
- **체크포인트 수**: 생성된 스냅샷 개수

### 메트릭 활용

- **예산 관리**: 작업별 비용 추적
- **효율 분석**: 비용이 큰 작업 파악
- **모델 비교**: 모델별 비용/사용량 비교
- **최적화**: 개선 여지가 있는 작업 발견
