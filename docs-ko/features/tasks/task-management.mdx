캐러티는 작업 히스토리를 관리할 수 있는 도구를 제공합니다. 시간이 지날수록 작업이 쌓이는데, 이 기능들은 생산성을 유지하는 데 핵심입니다.

## 작업 히스토리 접근

캐러티에서 작업 히스토리를 여는 방법은 여러 가지입니다. 버튼 클릭, 단축키, 명령 팔레트 등 편한 방식을 선택하세요.

작업 히스토리 접근 방법:

1. 캐러티 사이드바의 **"History" 버튼 클릭**
2. **명령 팔레트**에서 "캐러티: Show Task History" 검색
3. **키보드 단축키** (VS Code 설정에 따라 다름)

## 작업 히스토리 화면

작업 히스토리 화면은 작업을 보여주고 관리하는 중심 인터페이스입니다. 레이아웃, 검색, 정렬, 필터링을 통해 많은 작업을 효율적으로 탐색할 수 있습니다.

### 검색 및 필터

#### 검색 바
- 작업 전체에서 **퍼지 검색** 지원
- 프롬프트/응답/코드를 모두 검색
- 입력 즉시 결과 필터링
- 결과 내 일치 텍스트 강조 표시

#### 정렬 옵션
다음 기준으로 정렬할 수 있습니다:
- **최신순**(기본값)
- **오래된 순**
- **가장 비싼 작업** (API 비용)
- **토큰 사용량 최다**
- **가장 관련도 높은 결과** (검색 시)

#### 즐겨찾기 필터
- 별표 표시한 작업만 보기
- 중요한 작업을 빠르게 접근
- 검색과 결합해 정확한 필터링 가능

## 작업 액션

각 작업에는 다양한 작업을 수행하는 액션이 있습니다:

### 주요 액션

- **열기**: 작업을 다시 열어 채팅으로 복귀
- **재개**: 중단된 작업 이어서 진행
- **내보내기**: 대화를 마크다운으로 저장

### 관리 액션

- **즐겨찾기** ⭐: 별표를 눌러 중요한 작업 표시
- **삭제** 🗑️: 개별 작업 삭제(즐겨찾기는 보호됨)
- **복제**: 기존 작업을 기반으로 새 작업 생성

## ⭐ 작업 즐겨찾기

즐겨찾기 시스템은 중요한 작업을 보호하고 빠르게 접근할 수 있게 합니다.

### 즐겨찾기 사용법

**즐겨찾기 표시**
- 작업 옆 별표 아이콘 클릭
- 즐겨찾기 설정 시 별표가 채워짐
- 다시 클릭하면 해제

**보호 기능**
- 즐겨찾기 작업은 삭제로부터 보호
- 대량 삭제 시 즐겨찾기 자동 제외
- 명시적 확인으로 보호 해제 가능

**활용 사례**
- 참고용 구현 결과
- 성공적인 문제 해결 패턴
- 재사용 가능한 코드 스니펫
- 프로젝트 마일스톤
- 팀 학습용 예시

## 작업 메트릭

작업 메트릭으로 캐러티 사용량을 파악할 수 있습니다.

### 제공 메트릭

- **토큰 사용량**: 입력/출력 토큰 총합
- **API 비용**: 모델 가격 기반 비용 추정
- **체크포인트 수**: 생성된 파일 스냅샷 수

### 메트릭 활용

- **예산 추적**: 작업별 API 비용 모니터링
- **효율 분석**: 비용이 큰 작업 식별
- **모델 비교**: 모델별 비용 비교
- **최적화**: 비효율적인 작업 개선
