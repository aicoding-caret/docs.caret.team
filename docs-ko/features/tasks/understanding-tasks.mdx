## 작업이란?

대부분의 사용자는 Caret을 **작업(Task)**을 통해 사용합니다. 새로운 기능을 만들든, 버그를 고치든, 리팩터링을 하든, 코드베이스를 탐색하든 모든 상호작용은 하나의 작업 안에서 이루어집니다. 작업은 사용자가 입력한 **프롬프트**로 시작되며, 해당 작업의 전체 대화/코드 변경/명령 실행/의사결정이 하나의 세션으로 기록됩니다.

이 구조 덕분에 작업은 **정리되고, 추적 가능하며, 언제든 재개할 수** 있습니다. 각 작업은 독립된 컨텍스트를 유지하므로 여러 프로젝트를 동시에 진행해도 혼동이 없습니다. Caret의 작업 시스템은 사용자가 방향을 제시하면 Caret이 실행을 책임지는 협업형 개발 세션을 제공합니다.

### 핵심 특징

각 작업은 다음을 제공합니다:

- **고유 ID**: 작업마다 고유 식별자와 전용 저장 디렉터리
- **전체 대화 기록**: 모든 메시지/도구 호출/결과 보존
- **리소스 추적**: 토큰 사용량, API 비용, 실행 시간 기록
- **중단 및 재개**: VS Code 재시작 후에도 상태 유지
- **체크포인트**: Git 기반 스냅샷으로 파일 변경 추적
- **문서화**: 작업 내용을 Markdown으로 내보내 팀 문서화 가능
- **비용 관리**: 리소스 추적으로 사용량 파악

이 기능들은 Caret을 단순한 코딩 도구가 아닌 **작업의 전 생애주기를 이해하는 개발 에이전트**로 만들어 줍니다.

## 프롬프트로 작업 만들기

작업은 프롬프트로 시작됩니다. 결과 품질은 **요구사항을 얼마나 명확히 설명하느냐**에 크게 좌우됩니다.

### 좋은 프롬프트 구성 요소

보통 다음 요소가 포함되면 좋습니다:

- **목표**: 무엇을 달성하고 싶은지
- **컨텍스트**: 배경 정보와 제약 조건
- **요구사항**: 필요한 기능/동작/결과
- **선호사항**: 기술 스택, 코딩 스타일 등
- **예시**: 참고할 기준이나 샘플

<Note>
**프롬프트를 제대로 익히고 싶나요?**

[**Caret Learn**](https://caret.bot/learn)의 "Prompting" 모듈에서 구조화 프롬프트, 컨텍스트 최적화, 고급 프롬프트 패턴을 배울 수 있습니다.

좋은 프롬프트는 작업 완료 속도를 높이고, 정확도를 올리며, 반복 횟수를 줄여 더 나은 코드를 만듭니다.
</Note>

## 작업 실행 모드

Caret은 두 가지 모드로 작업을 수행합니다:

- **Plan 모드**: 정보 수집과 접근 전략 수립 (파일 변경 없음)
- **Act 모드**: 실제 구현 수행 (파일 수정, 명령 실행, 도구 사용)

→ **[Plan & Act 모드 알아보기](/features/plan-and-act)**

## 작업 리소스

작업은 다음 리소스를 사용하며 추적됩니다:

- **토큰 사용량**: 입력/출력 텍스트 규모
- **API 비용**: 모델과 토큰 사용량 기반 비용
- **시간**: 시작부터 완료까지의 시간
- **체크포인트**: 생성된 스냅샷 수

## 자주 쓰는 작업 패턴

### 코드 생성
```
정규식으로 이메일을 검증하는 TypeScript 함수를 만들어줘.
Jest 테스트를 추가하고 국제 도메인 같은 엣지 케이스를 처리해줘.
```

### 버그 수정
```
@terminal 제출 버튼을 클릭하면 앱이 크래시돼.
에러를 수정하고 적절한 에러 처리를 추가해줘.
```

### 리팩터링
```
@auth.ts의 인증 로직을 콜백 대신 async/await로 바꿔줘.
기존 기능은 유지해야 해.
```

### 기능 구현
```
설정 화면에 다크 모드 토글을 추가해줘.
기존 테마 컨텍스트를 사용하고 localStorage에 저장해줘.
```

## 작업 재개

Caret의 강력한 기능 중 하나는 **중단된 작업을 재개**할 수 있다는 점입니다.

### 작업이 중단되는 경우

- 긴 작업을 멈춘 경우
- 오류가 발생해 중단된 경우
- 다른 작업으로 전환한 경우

### 작업 재개 방법

1. 작업 기록에서 원하는 작업을 열기
2. Caret이 전체 대화를 로드
3. 체크포인트와 파일 상태를 비교
4. 중단 지점을 인지한 상태에서 계속 진행
5. 필요 시 추가 컨텍스트 제공

## 작업 컨텍스트 이해

작업은 생애주기 동안 다음 컨텍스트를 유지합니다:

- **대화 기록**: 이전 메시지와 응답
- **파일 변경**: 수정 내역과 순서
- **도구 결과**: 명령 출력과 작업 결과
- **체크포인트 상태**: 핵심 지점의 스냅샷

이를 통해 Caret은:
- 무엇이 이미 수행됐는지 이해
- 접근 방식을 일관되게 유지
- 지능적으로 작업을 재개
- 이전 시도를 학습

→ **[컨텍스트 관리 알아보기](/getting-started/understanding-context-management)**

작업 시스템을 이해하면 Caret을 훨씬 효과적으로 사용할 수 있습니다. 명확한 프롬프트와 작업 생애주기에 대한 이해가 핵심입니다.
