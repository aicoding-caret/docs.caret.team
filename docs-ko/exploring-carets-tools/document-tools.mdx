<Note>
이 도구들은 Caret 전용 기능입니다.
</Note>

Caret은 AI가 이미지와 문서를 직접 분석할 수 있는 강력한 도구를 제공합니다. 이러한 도구 덕분에 사용자가 파일을 직접 첨부하지 않아도 AI가 경로만으로 파일을 읽고 분석할 수 있습니다.

## 이미지 분석 도구 (analyze_image)

`analyze_image` 도구는 이미지 파일을 분석하여 내용을 설명하거나 특정 질문에 답변합니다.

### 사용 시나리오

- **UI/UX 검토**: 스크린샷에서 레이아웃 문제, 정렬 오류, 시각적 버그 확인
- **텍스트 추출**: 스크린샷, 에러 다이얼로그, 로그에서 텍스트 추출
- **코드 리뷰**: 코드 스크린샷 분석 및 패턴 식별
- **디자인 비교**: UI 구현과 디자인 목업 비교
- **에러 분석**: 에러 다이얼로그, 스택 트레이스 해석

### 지원 형식

- PNG, JPG, JPEG, WebP, GIF, AVIF, BMP, TIFF

### 사용 방법

```
사용자: 이 스크린샷(./screenshots/error.png)에서 에러 메시지를 추출해줘
AI: [analyze_image 도구 사용] → 이미지 분석 → 에러 메시지 추출 결과 제공
```

### 요구 사항

- **Caret 계정 로그인 필요**: 이미지 분석은 Caret API를 통해 처리됩니다
- **자동 이미지 최적화**: 대용량 이미지는 자동으로 최적화되어 전송됩니다

### 설정

설정 → 기능에서 **이미지 분석 모델**을 선택할 수 있습니다:
- Gemini 3.0 Flash Preview (기본값)
- Gemini 2.5 Flash

## 문서 읽기 도구 (read_document)

`read_document` 도구는 다양한 문서 형식을 읽어 텍스트를 추출합니다.

### Cline과의 비교

| 기능 | Cline | Caret |
|------|-------|-------|
| PDF 읽기 | 사용자 첨부 시에만 | **AI가 경로로 직접 읽기** |
| DOCX/XLSX 읽기 | 사용자 첨부 시에만 | **AI가 경로로 직접 읽기** |
| HWPX (한글) | ❌ 미지원 | ✅ **지원** |
| HWP 5.0 (구형 한글) | ❌ 미지원 | ✅ **지원** |
| PPTX | ❌ 미지원 | ✅ **지원** |

### 지원 형식

| 형식 | 확장자 | 설명 |
|------|--------|------|
| PDF | `.pdf` | Adobe PDF 문서 |
| Word | `.docx` | Microsoft Word 2007+ |
| Excel | `.xlsx` | Microsoft Excel 2007+ |
| PowerPoint | `.pptx` | Microsoft PowerPoint 2007+ |
| 한글 (신형) | `.hwpx` | 한컴오피스 한글 2014+ |
| 한글 (구형) | `.hwp` | 한컴오피스 한글 5.0 |
| Jupyter | `.ipynb` | Jupyter Notebook |

### 미지원 형식

다음 레거시 형식은 기술적 한계로 지원되지 않습니다:

| 형식 | 확장자 | 대안 |
|------|--------|------|
| PowerPoint 97-2003 | `.ppt` | `.pptx`로 변환 후 사용 |
| Word 97-2003 | `.doc` | `.docx`로 변환 후 사용 |
| Excel 97-2003 | `.xls` | `.xlsx`로 변환 후 사용 |

<Tip>
레거시 형식은 LibreOffice, Google 문서 도구, 또는 Microsoft Office에서 최신 형식으로 변환하세요.
</Tip>

### 사용 방법

```
사용자: 이 프로젝트의 docs/spec.pdf 분석해줘
AI: [read_document 도구 사용] → PDF 내용 추출 → 분석 결과 제공
```

### 보안

- **경로 검증**: Path Traversal 공격 방지를 위한 경로 정규화
- **파일 크기 제한**: 최대 50MB
- **형식 검증**: 지원되는 확장자만 허용

### 제한사항

- **이미지/차트 미포함**: 문서 내 이미지와 차트는 추출되지 않습니다
- **레이아웃 손실**: 표, 다단 레이아웃 등은 단순 텍스트로 변환됩니다

## 활용 팁

### 이미지 분석 활용

```
# UI 버그 확인
"./screenshots/ui-bug.png 스크린샷에서 레이아웃 문제가 있는지 확인해줘"

# 에러 메시지 분석
"이 에러 스크린샷(./error.png)에서 에러 원인을 분석해줘"

# 디자인 검토
"./mockup.png 디자인과 현재 구현이 일치하는지 비교해줘"
```

### 문서 분석 활용

```
# 사양서 분석
"docs/api-spec.pdf를 읽고 필요한 API 엔드포인트 목록을 정리해줘"

# 데이터 분석
"data/report.xlsx 파일의 내용을 요약해줘"

# 한글 문서 처리
"contracts/계약서.hwp 파일에서 주요 조항을 추출해줘"
```
