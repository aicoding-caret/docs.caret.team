# ^ 이미지 도구

<BrandName particle="of" /> AI는 **이미지를 생성**하고 **분석**할 수 있습니다.

## Cline과의 차이점

| 기능 | Cline | <BrandName /> |
|------|-------|-------|
| 이미지 생성 | 미지원 | **generate_image 도구로 지원** |
| 이미지 분석 (비전 미지원 모델) | 미지원 | **analyze_image 도구로 지원** |
| 비율/사이즈 설정 | 미지원 | **UI에서 설정 가능** |
| 참조 이미지 기반 생성 | 미지원 | **Image-to-Image 지원** |

## 이미지 생성

### 사용 예시

```
사용자: 귀여운 고양이 이미지 만들어줘
AI: [generate_image 도구 사용] → 이미지 생성 → assets/에 저장
```

### 설정 옵션

| 옵션 | 값 | 설명 |
|------|-----|------|
| **비율** | `16:9`, `9:16`, `4:3`, `3:4`, `1:1` | 이미지 가로세로 비율 |
| **사이즈** | `1K`, `2K`, `3K`, `4K` | 이미지 해상도 |

설정 위치: Settings > Image Generation

### 저장 위치

생성된 이미지는 프로젝트의 `assets/` 폴더에 저장됩니다:

```
assets/
├── img_abc123.png        # 생성된 이미지
└── img_abc123.md         # 메타데이터 (프롬프트, 설정 등)
```

## 이미지 분석

### 비전 모델 vs 텍스트 모델

| 모델 유형 | 이미지 분석 방법 |
|----------|----------------|
| **비전 모델** (GPT-4o, Claude 3.5 등) | `read_file`로 직접 분석 |
| **텍스트 모델** (o1, GLM-4 등) | `analyze_image` 도구 사용 |

### 사용 예시

```
# 비전 미지원 모델 (GLM-4.7 등)
사용자: [이미지 첨부] 이 이미지에 뭐가 있어?
AI: [analyze_image 도구 사용] → Gemini Flash로 분석 → 결과 반환
```

### 분석 모델 선택

텍스트 모델 사용 시 이미지 분석을 위한 모델을 선택할 수 있습니다:

- `gemini-2.5-flash` (기본값)
- `gemini-3.0-flash-preview`

설정 위치: Settings > Model Info > Image Analysis Model

## 인증 요구사항

| 도구 | 요구사항 |
|------|----------|
| `generate_image` | <BrandName /> 로그인 필요 |
| `analyze_image` | <BrandName /> 로그인 필요 (텍스트 모델만) |

## 설정

### Auto-approve 설정

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `generateImages` | `true` | 이미지 생성 도구 활성화 |
| `analyzeImages` | `true` | 이미지 분석 도구 활성화 |

설정 위치: Settings > Auto-approve

## 보안

### 이미지 분석 승인

| 파일 위치 | 동작 |
|-----------|------|
| 워크스페이스 내부 | ✅ 자동 승인 |
| 워크스페이스 외부 | ⚠️ 사용자 승인 필요 |

### 이미지 형식

지원되는 이미지 확장자:
- `.png`, `.jpg`, `.jpeg`, `.webp`
- `.gif`, `.avif`, `.bmp`, `.tiff`

## 알려진 제한사항

1. **인증 필수**: 이미지 도구는 <BrandName /> 로그인이 필요합니다
2. **단일 실행**: 이미지 도구는 한 번에 하나만 실행 가능합니다
3. **크기 제한**: 이미지 픽셀 최대 7500px
