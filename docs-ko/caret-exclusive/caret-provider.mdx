# <BrandName /> 제공자

<BrandName /> 제공자는 API 키 없이도 즉시 AI 코딩 어시스턴트를 사용할 수 있게 해주는 통합 인증 시스템입니다.

## 무엇인가요?

<BrandName particle="object" /> 처음 사용하는 많은 개발자들이 겪는 첫 번째 장벽은 바로 API 키 설정입니다. OpenAI나 Anthropic 같은 AI 제공자의 API 키를 발급받고, 결제 정보를 등록하고, 사용량을 관리하는 과정은 번거로울 수 있습니다.

<BrandName /> 제공자는 이러한 초기 진입 장벽을 제거합니다. <BrandName /> 계정 하나만 있으면, 별도의 API 키 설정 없이 즉시 AI 어시스턴트를 사용할 수 있습니다.

## Google Gemini 우선 지원

<BrandName /> 제공자의 첫 번째 지원 모델은 **Google Gemini**입니다. Gemini는 강력한 코딩 능력을 가지고 있으면서도, 상대적으로 저렴한 가격으로 제공되어 초보자와 개인 개발자에게 적합합니다.

## 무료 크레딧

<BrandName /> 제공자를 처음 사용하는 모든 사용자에게 **무료 크레딧**을 제공합니다. 이를 통해 실제 비용 부담 없이 <BrandName particle="of" /> 기능을 충분히 테스트해볼 수 있습니다.

## 이미지 생성 도구 (<BrandName /> 계정 기반)

<BrandName /> 계정으로 로그인하면 이미지 생성 도구를 사용할 수 있습니다. 이 기능은 <BrandName /> 토큰으로 인증되며, 생성 결과가 작업공간에 자동 저장됩니다.

### 어떻게 동작하나요?

1. <BrandName particle="at" />게 이미지 생성 요청을 합니다. (예: "대시보드 일러스트를 만들어줘")
2. <BrandName particle="subject" /> 이미지 생성 API를 호출해 결과를 생성합니다.
3. 결과 파일이 작업공간의 `.agents/generated-assets/` 폴더에 저장됩니다.

### 저장 규칙

이미지 생성 결과는 요청 ID 기준으로 다음 두 파일이 생성됩니다.

- `.agents/generated-assets/<request_id>.<ext>`: 실제 이미지 파일
- `.agents/generated-assets/<request_id>.md`: 메타데이터(프롬프트/링크) 기록

### 레퍼런스 이미지 활용

이미지 생성 시 텍스트 프롬프트뿐 아니라 **이미지도 함께 첨부**해 요청할 수 있습니다. 사용자가 **특정 이미지를 레퍼런스로 지정**하면 스타일/구도/캐릭터의 **일관성을 유지하면서 이미지 변형·편집**을 진행할 수 있습니다.  
레퍼런스 이미지를 지정하지 않으면, 선택한 이미지 스코프가 있거나 기본 동작에 따라 입력 이미지가 자동 참조될 수 있습니다.

### 실무 팁

- 이미지 파일과 메타데이터가 저장되므로, 저장소에 포함할지 여부를 팀 기준에 맞게 결정하세요.
- 프롬프트가 `.md`에 그대로 기록되므로 민감한 정보는 포함하지 않는 것이 좋습니다.
- 필요하면 <BrandName particle="at" />게 생성된 이미지/메타데이터를 **프로젝트 내 특정 폴더로 이동**하도록 지시해 체계적으로 관리할 수 있습니다.
- 이 흐름을 이용하면 **문서 기반 프롬프트 관리와 리소스 편집**이 가능하며, 세계관/캐릭터 가이드 같은 자료와 프롬프트를 함께 관리하면서 콘텐츠 제작이나 서비스 리소스 제작에 활용할 수 있습니다.
