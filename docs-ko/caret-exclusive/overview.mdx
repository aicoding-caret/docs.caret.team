# Caret만의 특별한 기능들

Caret은 단순히 Cline을 리브랜딩한 것이 아닙니다. Cline의 견고한 기반 위에 실질적인 확장 기능을 추가하여 더 나은 코딩 경험을 제공합니다.

## Caret이 추가로 제공하는 기능

Caret은 Cline의 모든 기능을 완벽히 지원하면서, 다음과 같은 확장 기능을 제공합니다:

### 🔄 듀얼 모드 시스템
Caret은 **Caret 모드**와 **Cline 모드**를 원클릭으로 전환할 수 있습니다. Caret 모드에서는 Agent/Chatbot 방식의 직관적인 대화를, Cline 모드에서는 기존 Plan/Act 방식을 그대로 사용할 수 있어 상황에 맞는 최적의 워크플로우를 선택할 수 있습니다.

### 추가 확장 기능
- **🎭 페르소나 시스템** — 나만의 AI 이름과 아바타를 설정하여 개인화된 코딩 파트너 경험
- **⌨️ 프롬프트 히스토리** — 터미널처럼 ↑↓ 키로 이전 입력 빠르게 재사용
- **📋 에이전트 표준화(AAIF SoT)** — `.agents/context` 기반의 체계적인 규칙 관리
- **🛠️ 스킬 시스템** — 이중 디렉토리 아키텍처로 재사용 가능한 지침 세트 제공
- **🌍 완전한 다국어 UI** — 한국어, 영어, 일본어, 중국어, 프랑스어, 독일어, 러시아어 7개 언어 지원
- **🚀 프로바이더 자동 조회** — LiteLLM 연동으로 모델 목록 자동 업데이트
- **🇰🇷 한국 AI 프로바이더** — Upstage, Naver 등 한국 AI 기업 모델 지원

## Caret 독점 기능 둘러보기

### 🎭 페르소나 시스템

AI의 이름과 아바타를 커스터마이징하여 나만의 코딩 파트너를 만드세요.

- 5가지 기본 페르소나 제공 (Caret, 오사랑, 마도베 이치카, 시안 맥킨, 탄도 우분투)
- 커스텀 이미지 업로드 (일반/생각중 2가지)
- persona.md 파일로 성격 정의

[자세히 보기 →](/ko/caret-exclusive/persona-system)

### 🔄 듀얼 모드 시스템

작업에 따라 최적화된 두 가지 모드를 전환하세요:

- **Agent 모드**: 자율적인 AI 파트너 (대화 전환 유연)
- **Chatbot 모드**: 개발자 주도 조언자 (AI 통제)

*참고: Cline은 Plan/Act 단일 방식만 지원*

[자세히 보기 →](/ko/caret-exclusive/dual-prompt-modes)

### ⌨️ 프롬프트 히스토리 시스템

터미널과 동일한 ↑↓ 방향키로 이전 입력을 빠르게 재사용하세요.

- 영구 저장 (VS Code 재시작 후에도 유지)
- 최대 1000개 히스토리
- 중복 자동 제거

### 📋 고급 규칙 시스템 (AAIF SoT)

AI와 사람이 읽는 문서의 목적이 다르다는 점을 인정하고, **이중 디렉토리 아키텍처**로 명확하게 분리합니다:

- **`.agents/`** — AI용 (영어, 토큰 최적화, JSON/YAML)
- **`.users/`** — 사람용 (네이티브 언어, 상세 설명, Markdown)

**핵심 기능**:
- **단일 SoT**: `.agents/context/agents-rules.json`을 규칙의 단일 진실 원천으로 유지
- **계층 탐색**: `AGENTS.md`를 재귀적으로 탐색하여 디렉토리별 규칙 적용
- **온디맨드 워크플로우**: 필요한 워크플로우만 로드하여 토큰 효율 극대화
- **`/init` 스캐폴드**: 신규 프로젝트에 표준 구조를 자동 생성 (기존 파일 보존)
- **레거시 폴백**: 기존 규칙 경로도 자동 감지하여 마이그레이션 안내

[자세히 보기 →](/ko/caret-exclusive/knowledge-parity-system)

### 🛠️ 스킬 시스템 (v0.4.5 신규)

재사용 가능한 지침 세트로 Caret의 능력을 확장하세요:

- **온디맨드 로딩**: 필요한 스킬만 로드하여 컨텍스트 효율 극대화
- **도메인별 패키징**: `.agents/skills/`에 전문 지식을 캡슐화
- **전역/프로젝트 분리**: 글로벌 스킬과 프로젝트 스킬을 분리 관리
- **자동/수동 활성화**: 조건에 따른 자동 활성화 및 수동 토글 지원

[자세히 보기 →](/ko/features/skills)

### 🤝 AI-개발자 지식 동기화

Caret의 AI는 어떻게 개발자와 동일한 지식을 공유하여 높은 신뢰도를 확보할까요? 그 배경이 되는 오픈소스 개발 워크플로우를 소개합니다. 이 시스템은 Caret AI의 투명성과 신뢰성의 근간이 됩니다.

[자세히 보기 →](/ko/caret-exclusive/knowledge-parity-system)

### 🌍 완전한 다국어 지원

UI와 문서가 모두 번역된 진정한 다국어 지원:

- **7개 언어 지원**: 한국어, 영어, 일본어, 중국어, 프랑스어, 독일어, 러시아어
- 모든 설정과 메시지 번역
- 언어별 문서 사이트

[자세히 보기 →](/ko/caret-exclusive/multilingual-ui)

- 비공개 커스터마이징 프로젝트
- 화이트라벨 솔루션

**문의**: support@caret.team

### 🚀 향상된 프로바이더 설정

Caret은 프로바이더 설정 경험을 대폭 개선했습니다.

**Sovereign Cloud 원칙**: 프로바이더별 국가 플래그를 표시하여 데이터 주권을 한눈에 파악할 수 있습니다:
- 🇺🇸 미국: Anthropic, OpenAI, xAI, Groq 등
- 🇰🇷 한국: Caret, Upstage, Naver Cloud, BizRouter
- 🇨🇳 중국: Qwen, DeepSeek, Doubao, Moonshot 등
- 🇫🇷 프랑스: Mistral
- 🇩🇪 독일: SAP AI Core
- 🇷🇺 러시아: Nebius
- ☁️ 글로벌 클라우드: Bedrock, Vertex, Gemini
- 💻 로컬: Ollama, LM Studio, VS Code LM

**자동 모델 조회**: LiteLLM API와 통합하여 프로바이더 선택 시 지원 모델 목록을 자동으로 표시합니다. 새 모델이 출시되어도 Caret 업데이트 없이 자동 반영됩니다.

**100개 이상의 프로바이더 지원**: OpenAI, Anthropic, Google Gemini부터 Cohere, AI21 Labs, Hugging Face까지 다양한 프로바이더를 지원합니다.

[자세히 보기 →](/ko/caret-exclusive/enhanced-provider-setup)

## 왜 Caret을 선택해야 할까요?

### 즉시 시작 가능

무료 Gemini 크레딧으로 API 키 설정 없이 바로 코딩을 시작할 수 있습니다 (출시 예정).

### 터미널과 일관된 UX

프롬프트 히스토리 시스템으로 bash/zsh와 동일한 ↑↓ 키 경험을 제공합니다.

### 당신만의 AI

페르소나 시스템으로 AI의 이름과 아바타를 원하는 대로 커스터마이징하세요.

### 유연한 작업 방식

Agent 모드는 유연한 대화, Chatbot 모드는 개발자 주도 작업에 최적화되어 있습니다.

### 모국어 지원

한국어를 포함한 7개 언어의 완전한 지원으로 더 편안하게 사용하세요.
