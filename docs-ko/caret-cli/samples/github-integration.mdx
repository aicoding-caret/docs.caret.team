# GitHub 연동 샘플

AI로 GitHub 이슈 분석을 자동화하세요. 이슈 댓글에 `@caret`를 멘션하면, 파일을 읽고 코드를 분석해 실행 가능한 인사이트를 제공하는 자율 분석이 GitHub Actions에서 자동으로 실행됩니다.


<Note>
**Caret CLI가 처음인가요?** 이 샘플은 Caret CLI 기본 사용법을 알고 있고 [설치 가이드](/caret-cli/installation)를 완료했다는 전제입니다. 처음이라면 더 간단한 [GitHub RCA 샘플](./github-issue-rca)부터 시작하는 것을 권장합니다.
</Note>

이제 저장소에 설정해 보겠습니다.

## 사전 준비

시작하기 전에 다음 항목이 필요합니다:

- **Caret CLI 지식** - [설치 가이드](/caret-cli/installation) 완료 및 기본 사용법 이해
- **GitHub 저장소** - Actions/Secrets 설정을 위한 관리자 권한
- **GitHub Actions 이해** - 워크플로우/CI-CD 기본 개념
- **API 프로바이더 계정** - OpenRouter, Anthropic 등 API 키 보유

## 설정

### 1. 워크플로우 파일 복사

샘플 워크플로우 파일을 저장소로 복사합니다. GitHub Actions가 인식할 수 있도록 `.github/workflows/` 디렉터리에 배치해야 합니다. 여기서는 `caret-responder.yml`로 이름을 지정합니다.

```bash
# 저장소 루트에서 실행
mkdir -p .github/workflows
curl -o .github/workflows/caret-responder.yml https://raw.githubusercontent.com/caret/caret/main/src/samples/cli/github-integration/caret-responder.yml
```

또는 아래 전체 내용을 `.github/workflows/caret-responder.yml`에 직접 붙여넣을 수 있습니다:

<Accordion title="caret-responder.yml 전체 보기">
```yaml
name: Caret Issue Assistant

on:
  issue_comment:
    types: [created, edited]

permissions:
  issues: write

jobs:
  respond:
    runs-on: ubuntu-latest
    environment: caret-actions
    steps:
      - name: Check for @caret mention
        id: detect
        uses: actions/github-script@v7
        with:
          script: |
            const body = context.payload.comment?.body || "";
            const isPR = !!context.payload.issue?.pull_request;
            const hit = body.toLowerCase().includes("@caret");
            core.setOutput("hit", (!isPR && hit) ? "true" : "false");
            core.setOutput("issue_number", String(context.payload.issue?.number || ""));
            core.setOutput("issue_url", context.payload.issue?.html_url || "");
            core.setOutput("comment_body", body);

      - name: Checkout repository
        if: steps.detect.outputs.hit == 'true'
        uses: actions/checkout@v4

      # Node v20 is needed for Caret CLI on GitHub Actions Linux
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Setup Caret CLI
        if: steps.detect.outputs.hit == 'true'
        run: |
          # Install the Caret CLI
          sudo npm install -g caret

      - name: Create Caret Instance
        if: steps.detect.outputs.hit == 'true'
        env:
          OPENROUTER_API_KEY: ${{ secrets.OPENROUTER_API_KEY }}
          CLINE_DIR: ${{ runner.temp }}/caret
        run: |
          # Create instance and capture output
          INSTANCE_OUTPUT=$(caret instance new 2>&1)
          
          # Parse address from output (format: "  Address: 127.0.0.1:36733")
          CLINE_ADDRESS=$(echo "$INSTANCE_OUTPUT" | grep "Address:" | grep -oE '([0-9]{1,3}\.){3}[0-9]{1,3}:[0-9]+')
          echo "CLINE_ADDRESS=$CLINE_ADDRESS" >> $GITHUB_ENV
          
          # Configure API key
          caret config set open-router-api-key=$OPENROUTER_API_KEY --address $CLINE_ADDRESS -v

      - name: Download analyze script
        if: steps.detect.outputs.hit == 'true'
        run: |
          export GITORG="YOUR-GITHUB-ORG"
          export GITREPO="YOUR-GITHUB-REPO"

          curl -L https://raw.githubusercontent.com/${GITORG}/${GITREPO}/refs/heads/main/git-scripts/analyze-issue.sh -o analyze-issue.sh
          chmod +x analyze-issue.sh

      - name: Run analysis
        if: steps.detect.outputs.hit == 'true'
        id: analyze
        env:
          ISSUE_URL: ${{ steps.detect.outputs.issue_url }}
          COMMENT: ${{ steps.detect.outputs.comment_body }}
          CLINE_ADDRESS: ${{ env.CLINE_ADDRESS }}
        run: |
          set -euo pipefail
          
          RESULT=$(./analyze-issue.sh "${ISSUE_URL}" "Analyze this issue. The user asked: ${COMMENT}" "$CLINE_ADDRESS")
          
          {
            echo 'result<<EOF'
            printf "%s\n" "$RESULT"
            echo 'EOF'
          } >> "$GITHUB_OUTPUT"

      - name: Post response
        if: steps.detect.outputs.hit == 'true'
        uses: actions/github-script@v7
        env:
          ISSUE_NUMBER: ${{ steps.detect.outputs.issue_number }}
          RESULT: ${{ steps.analyze.outputs.result }}
        with:
          script: |
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: Number(process.env.ISSUE_NUMBER),
              body: process.env.RESULT || "(no output)"
            });
```
</Accordion>

<Warning>
**커밋 전에 반드시 워크플로우를 수정하세요!**

`.github/workflows/caret-responder.yml`을 열고 “Download analyze script” 단계에서 분석 스크립트가 있는 GitHub 조직/리포지토리를 지정해야 합니다:

```yaml
export GITORG="YOUR-GITHUB-ORG"      # 반드시 변경
export GITREPO="YOUR-GITHUB-REPO"    # 반드시 변경
```

**예시:** 저장소가 `github.com/acme/myproject`라면 다음과 같이 설정합니다:
```yaml
export GITORG="acme"
export GITREPO="myproject"
```

이 설정은 3단계에서 커밋한 분석 스크립트를 어디서 다운로드할지 알려줍니다.
</Warning>

워크플로우는 새 이슈/업데이트된 이슈를 감지하고, `@caret` 멘션을 확인한 뒤, Caret CLI 인스턴스를 시작해 분석 결과를 댓글로 응답합니다.

1. GitHub 저장소로 이동

### 3. 분석 스크립트 추가

`github-issue-rca` 샘플의 분석 스크립트를 저장소에 추가합니다. **먼저 저장소 루트에 `git-scripts` 디렉터리를 생성해야 합니다.** 아래 중 하나를 선택하세요:

**옵션 A: 바로 다운로드(권장)**

```bash
# 저장소 루트에서 디렉터리 생성 후 스크립트 다운로드
mkdir -p git-scripts
curl -o git-scripts/analyze-issue.sh https://raw.githubusercontent.com/caret/caret/main/src/samples/cli/github-issue-rca/analyze-issue.sh
chmod +x git-scripts/analyze-issue.sh
```

**옵션 B: 수동 복사**

디렉터리와 파일을 직접 만든 뒤 스크립트를 붙여넣습니다:

```bash
# 저장소 루트에서
mkdir -p git-scripts
# 원하는 편집기로 파일 생성
nano git-scripts/analyze-issue.sh  # 또는 vim, code 등
```

<Accordion title="analyze-issue.sh 전체 스크립트 보기">
```bash
#!/bin/bash
# Analyze a GitHub issue using Caret CLI

if [ -z "$1" ]; then
    echo "Usage: $0 <github-issue-url> [prompt] [address]"
    echo "Example: $0 https://github.com/owner/repo/issues/123"
    echo "Example: $0 https://github.com/owner/repo/issues/123 'What is the root cause of this issue?'"
    echo "Example: $0 https://github.com/owner/repo/issues/123 'What is the root cause of this issue?' 127.0.0.1:46529"
    exit 1
fi

# Gather the args
ISSUE_URL="$1"
PROMPT="${2:-What is the root cause of this issue?}"
if [ -n "$3" ]; then
    ADDRESS="--address $3"
fi

# Ask Caret for its analysis, showing only the summary
caret -y "$PROMPT: $ISSUE_URL" --mode act $ADDRESS -F json | \
    sed -n '/^{/,$p' | \
    jq -r 'select(.say == "completion_result") | .text' | \
    sed 's/\\n/\n/g'
```

스크립트를 붙여넣은 뒤 실행 권한을 부여하세요:
```bash
chmod +x git-scripts/analyze-issue.sh
```
</Accordion>

이 분석 스크립트는 Caret에 프롬프트를 실행하게 하고, 요약 결과만 추출해 이슈 답글로 사용합니다.

### 4. 커밋 및 푸시

```bash
git add .github/workflows/caret-responder.yml
git add git-scripts/analyze-issue.sh
git commit -m "Add Caret issue assistant workflow"
git push
```

## 사용법

설정 후, 이슈 댓글에 `@caret`를 멘션하면 됩니다:

```
@caret what's causing this error?

@caret analyze the root cause

@caret what are the security implications?
```

GitHub Actions는 다음을 수행합니다:
1. `@caret` 멘션 감지
2. Caret CLI 인스턴스 시작
3. 분석 스크립트 다운로드
4. act 모드 + yolo(완전 자동)로 이슈 분석
5. Caret 분석 결과를 댓글로 게시

**참고**: 이 워크플로우는 이슈 댓글에만 반응하며 PR 댓글에는 동작하지 않습니다.

## 작동 원리

워크플로우(`caret-responder.yml`)는 다음 과정을 수행합니다:

1. 이슈 댓글 생성/수정 시 트리거
2. `@caret` 멘션 감지(대소문자 무시)
3. npm으로 Caret CLI 전역 설치
4. `caret instance new`로 인스턴스 생성
5. `caret config set open-router-api-key=... --address ...`로 인증 구성
6. `github-issue-rca` 샘플에서 분석 스크립트 다운로드
7. 인스턴스 주소로 분석 실행
8. 결과를 댓글로 게시
