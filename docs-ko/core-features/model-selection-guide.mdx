새로운 모델이 계속 출시되므로, 이 가이드는 현재 Caret과 잘 맞는 모델 선택에 집중합니다. 환경이 바뀌면 지속적으로 업데이트합니다.

<Callout type="tip">
**모델 선택이 처음인가요?** 모델 비교와 컨텍스트 설정을 함께 다루는 [모델 비교 안내](/ko/model-config/model-comparison)와 [컨텍스트 윈도우 가이드](/ko/model-config/context-windows)를 먼저 참고해 보세요.
</Callout>

## AI 모델이란?

AI 모델은 Caret을 움직이는 “두뇌”라고 생각하면 됩니다. Caret에게 코드를 작성하거나, 버그를 수정하거나, 리팩터링을 요청하면 실제로 이해하고 응답을 생성하는 주체가 바로 모델입니다.

**핵심 포인트:**
- **모델은 자연어와 코드를 이해하도록 학습된 AI 시스템**입니다
- **모델마다 강점이 다릅니다** 복잡한 추론에 강한 모델도 있고, 속도나 비용을 우선하는 모델도 있습니다
- **Caret이 사용할 모델을 직접 선택합니다** 작업 유형에 맞는 전문가를 고르는 것과 같습니다
- **모델은 API 프로바이더를 통해 접근합니다** Anthropic, OpenAI, OpenRouter 등이 모델을 호스팅합니다

**왜 중요한가요?** 선택한 모델은 Caret의 성능, 응답 품질, 속도, 비용에 직접 영향을 줍니다. 고급 모델은 복잡한 리팩터링에 강하지만 비용이 높을 수 있고, 예산형 모델은 일상적인 작업에 효율적입니다.

## Caret에서 모델 선택하기

아래 5단계로 원하는 모델을 설정할 수 있습니다.

### 1단계: Careti 설정 열기

먼저 Caret의 설정 패널에 접근합니다.

**설정을 여는 두 가지 방법:**
- **빠른 방법**: Careti 채팅 UI 우측 상단의 **톱니바퀴(⚙️)** 클릭
- **명령 팔레트**: **Cmd/Ctrl + Shift + P** → "Careti: Open Settings" 입력

<Frame>
  <img src="https://storage.googleapis.com/cline_public_images/step1-config.png" alt="Careti Settings Panel" />
</Frame>

설정 패널이 열리면 상단에 "API Provider" 항목이 보입니다.

<Note>
설정 패널은 마지막 구성을 기억하므로 한 번 설정하면 반복할 필요가 없습니다.
</Note>

### 2단계: API 프로바이더 선택

드롭다운에서 선호하는 AI 프로바이더를 선택합니다.


<Frame>
  <img src="https://storage.googleapis.com/cline_public_images/step2-provider.png" alt="Careti Settings Panel" />
</Frame>

**인기 프로바이더 한눈에 보기:**

| 프로바이더 | 추천 용도 | 비고 |
|----------|----------|-------|
| **Careti** | 가장 쉬운 시작 | API 키 없이 사용 가능, stealth 모델 포함 다수 모델 접근 |
| **OpenRouter** | 가성비 | 다양한 모델과 경쟁력 있는 가격 |
| **Anthropic** | 안정성 | Claude 모델, 도구 사용 안정성 높음 |
| **OpenAI** | 최신 기술 | GPT 모델 |
| **Google Gemini** | 대형 컨텍스트 | Google 모델 |
| **AWS Bedrock** | 엔터프라이즈 | 고급 기능 |
| **Ollama** | 프라이버시 | 로컬 실행 가능 |

Cerebras, Vertex AI, Azure 등 더 많은 옵션은 [프로바이더 설정 가이드](/ko/provider-config/anthropic)에서 확인하세요.

<Info>
**처음 사용자 추천:** **Careti** 프로바이더부터 시작하세요. API 키 관리가 필요 없고 여러 모델에 즉시 접근할 수 있으며, 파트너 프로바이더를 통한 무료 추론이 제공될 때도 있습니다.
</Info>

### 3단계: API 키 추가 (또는 로그인)

선택한 프로바이더에 따라 다음 단계가 달라집니다.

#### **Careti** 프로바이더를 선택한 경우:

- **API 키가 필요 없습니다.** Careti 계정으로 로그인만 하면 됩니다
- 안내에 따라 **Sign In** 버튼 클릭
- [caret.team](https://careti.ai)으로 이동해 인증
- 인증 후 IDE로 돌아오기

#### 다른 프로바이더를 선택한 경우:

선택한 프로바이더에서 API 키를 발급받아야 합니다:

1. **프로바이더 사이트에서 API 키 발급:**
   - **Anthropic**: [console.anthropic.com](https://console.anthropic.com/)
   - **OpenRouter**: [openrouter.ai/keys](https://openrouter.ai/keys)
   - **OpenAI**: [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
   - **Google**: [aistudio.google.com/apikey](https://aistudio.google.com/apikey)
   - **기타**: [프로바이더 설정 가이드](/ko/provider-config/anthropic) 참고

2. **새 API 키 생성**

3. **API 키 복사**

4. Careti 설정의 **"API Key"** 필드에 붙여넣기

5. **자동 저장** - 키는 에디터의 시크릿 스토리지에 안전하게 저장됩니다

<Frame>
  <img src="https://storage.googleapis.com/cline_public_images/step3-API.png" alt="Careti API Selection" />
</Frame>

<Warning>
**대부분의 프로바이더는 결제가 필요합니다.** API 키 발급 전에 결제 정보를 요구하는 경우가 많으며, 일반적으로 작업당 $0.01-$0.10 수준으로 사용한 만큼만 비용이 발생합니다.
</Warning>

### 4단계: 모델 선택

API 키를 입력하거나 로그인하면 **"Model"** 드롭다운이 활성화됩니다.

<Frame>
  <img src="https://storage.googleapis.com/cline_public_images/step4-model.png" alt="Careti Model Selection" />
</Frame>

**빠른 모델 선택 가이드:**

| 우선순위 | 추천 모델 | 이유 |
|---------------|-------------------|-----|
| **최대 안정성** | Claude Sonnet 4.5 | 도구 사용 안정성이 가장 높고 복잡한 작업에 강함 |
| **가성비** | DeepSeek V3 또는 Qwen3 Coder | 예산 대비 뛰어난 성능 |
| **최고 속도** | Cerebras의 Qwen3 Coder | 응답 속도가 가장 빠름 |
| **로컬 실행** | Ollama 모델 | 완전한 프라이버시, 인터넷 불필요 |
| **최신 기능** | GPT-5 | OpenAI의 최신 기능 |

어떤 모델을 골라야 할지 고민된다면 **Claude Sonnet 4.5**(안정성) 또는 **DeepSeek V3**(가성비)부터 시작하세요.

<Tip>
대화 기록을 잃지 않고 언제든 모델을 변경할 수 있습니다. 작업 유형별로 다양한 모델을 시험해 보세요.
</Tip>

아래 [모델 비교 표](#current-top-models)를 참고하면 스펙과 가격을 더 자세히 확인할 수 있습니다.

## 현재 인기 모델 {#current-top-models}

현재 가장 많이 선택되는 모델은 다음과 같습니다:

- **안정성:** Claude Sonnet 4.5  
- **가성비:** DeepSeek V3 또는 Qwen3 Coder  
- **속도:** Cerebras의 Qwen3 Coder  
- **로컬:** Ollama 호스팅 모델  
- **최신 기능:** GPT-5

### 5단계: Careti 시작하기

**설정이 끝났습니다!** 이제 Caret으로 코딩을 시작해 보세요:

1. **Careti 채팅창에 요청 입력**
   - 예: "Create a React component for a login form"
   - 예: "Debug this TypeScript error"
   - 예: "Refactor this function to be more efficient"

2. **Enter** 또는 전송 아이콘 클릭

## 올바른 모델을 고르는 기준

모델 선택은 여러 요소의 균형입니다. 아래 프레임워크를 참고해 최적의 모델을 찾으세요.

<Note>
**프로 팁**: Plan 모드와 Act 모드에 서로 다른 모델을 설정해 보세요. 예를 들어 계획 단계에는 저렴한 모델, 구현 단계에는 고급 모델을 사용하는 방식이 효율적입니다.
</Note>

### 주요 선택 요소

| 요소 | 고려 사항 | 추천 |
|--------|------------------|----------------|
| **작업 복잡도** | 단순 수정 vs 복잡 리팩터링 | 루틴 작업은 예산형, 복잡 작업은 프리미엄 |
| **예산** | 월간 지출 규모 | $10-$30: 예산형, $30-$100: 중간, $100+: 프리미엄 |
| **컨텍스트 윈도우** | 프로젝트 규모/파일 수 | 소형: 32K-128K, 중형: 128K-200K, 대형: 400K+ |
| **속도** | 응답 시간 요구 | 대화형은 빠른 모델, 백그라운드는 추론형 가능 |
| **도구 신뢰성** | 복잡한 작업 수행 | Claude가 도구 사용에 강함. 워크플로우에 맞게 테스트 필요 |
| **프로바이더** | 접근성/가격 | OpenRouter: 선택 폭 넓음, Direct: 빠름/안정적, Local: 프라이버시 |

## 오픈 소스 vs 클로즈드 소스

### 오픈 소스의 장점
- **여러 프로바이더의 호스팅 경쟁**
- 경쟁으로 인한 **저렴한 가격**
- **프로바이더 선택** - 하나가 다운되면 다른 곳으로 전환 가능
- **빠른 혁신 속도**

### 사용 가능한 오픈 소스 모델
- **Qwen3 Coder** (Apache 2.0)
- **Z AI GLM 4.5** (MIT)
- **Kimi K2** (오픈 소스)
- **DeepSeek 시리즈** (다양한 라이선스)
