**가장 많이 적용되는 해결책부터 정리했습니다:**

- **bash로 전환** (대부분의 문제 해결)

  1. <BrandName particle="from" /> 채팅 우측 상단 **Settings** 버튼 클릭
  2. **Settings** 창 좌측 메뉴에서 **"Terminal Settings"** 탭 클릭
  3. **"Default Terminal Profile"**에서 **"bash"** 선택

- **타임아웃 증가**

  1. <BrandName particle="from" /> 채팅 우측 상단 **Settings** 버튼 클릭
  2. **Settings** 창 좌측 메뉴에서 **"Terminal Settings"** 탭 클릭
  3. "Shell integration timeout (seconds)"에 **"10"** 입력

- **터미널 재사용 비활성화**
  1. <BrandName particle="from" /> 채팅 우측 상단 **Settings** 버튼 클릭
  2. **Settings** 창 좌측 메뉴에서 **"Terminal Settings"** 탭 클릭
  3. **"Enable aggressive terminal reuse"**를 찾아 **체크 해제**

## 플랫폼별 해결 방법

### macOS + Oh-My-Zsh

```bash
# VSCode용 최소 설정 파일 생성
echo 'export TERM=xterm-256color' > ~/.zshrc-vscode
echo 'export PAGER=cat' >> ~/.zshrc-vscode
```

### Windows PowerShell

```powershell
# 관리자 권한으로 실행
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### WSL

- WSL에서 폴더 열기: `code .`
- <BrandName /> 설정의 **"Terminal Settings"**에서 **"WSL Bash"** 선택
- "Shell integration timeout (seconds)"를 **15**로 증가
