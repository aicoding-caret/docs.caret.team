# Caret 컨텍스트 시스템

Caret는 Cline Memory Bank 방식을 **완전 대체**했습니다.  
대신 **`AGENTS.md` + `.agents/context` + `caret-docs` + `work-logs`**로 프로젝트 컨텍스트를 운영합니다.

## 빠른 시작

1. **`/init`로 스캐폴드 생성**  
   `AGENTS.md`와 `.agents/context` 기본 구조를 자동으로 준비합니다.
2. **작업 시작 시 필수 문서 확인**  
   `AGENTS.md` → `.agents/context/caret-rules.json` 순으로 확인합니다.
3. **워크플로는 온디맨드 로딩**  
   필요한 워크플로만 `.agents/context/workflows/`에서 불러옵니다.
4. **개발자 문서와 진행 기록 유지**  
   변경 사항은 `caret-docs/`와 `work-logs/`에 기록합니다.

## 핵심 구성 요소

| 구성 | 역할 | 비고 |
| --- | --- | --- |
| `AGENTS.md` | 진입점/메타 규칙 | 로드 순서·언어·금지사항 |
| `.agents/context` | AI 규칙 SoT | 워크플로/아톰 포함 |
| `.agents/context/workflows` | 온디맨드 워크플로 | 필요할 때만 로딩 |
| `caret-docs/` | 개발자 문서 | AI 규칙과 1:1 동등 |
| `work-logs/` | 진행/결정 기록 | 서브모듈, 향후 `.work-logs/` 계획 |

## Caret가 Memory Bank를 대체한 이유 (차별점)

- **국제 표준(AAIF) 기반**: 규칙 경로와 로딩을 표준화했습니다.  
- **온디맨드 규칙 로딩**: 필요 문서만 읽어 컨텍스트를 최소화합니다.  
- **AI/개발자 지식 동등성(Parity)**: AI와 개발자가 동일 문서를 봅니다.  
- **진행 기록 분리**: 상태/결정은 `work-logs`에 기록해 흐름을 명확히 합니다.

## 기존 Memory Bank 매핑

| Memory Bank | Caret 대체 경로 |
| --- | --- |
| `projectbrief.md` | `caret-docs/development/index.md` + 기능 개요 문서 |
| `productContext.md` | `caret-docs/features.en/**` |
| `systemPatterns.md` | `.agents/context/workflows/atoms/**` + 아키텍처 문서 |
| `techContext.md` | `caret-docs/development/**` |
| `activeContext.md`/`progress.md` | `work-logs/**` (마스터/서브 문서) |

> **중요**: `memory-bank/` 폴더는 사용하지 않습니다. 기존 파일은 위 구조로 이동하세요.

## 작업 흐름

### Plan 모드
1. `AGENTS.md` → `.agents/context/caret-rules.json` 순서로 확인
2. 작업 유형에 맞는 워크플로만 로딩
3. `caret-docs/`에서 기능/가이드 확인
4. 계획 문서를 작성하고 공유

### Act 모드
1. 필요한 규칙/워크플로만 로딩
2. 구현 진행
3. 변경사항을 `caret-docs/`와 `work-logs/`에 기록

## 업데이트 규칙

- **규칙/프로세스 변경** → `.agents/context` + `caret-docs/` 동시 갱신
- **기능/요구사항 변경** → `caret-docs/features.en/**` 갱신
- **진행/결정 기록** → `work-logs/**`에 누적
- **보안** → 토큰/비밀정보는 문서에 기록하지 않음
