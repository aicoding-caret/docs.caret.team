관리자는 호스팅된 관리자 콘솔에서 LiteLLM을 **조직 전체 LLM 공급자**로 설정할 수 있습니다. LiteLLM 프록시를 통해 다양한 모델을 하나의 인터페이스로 제공합니다.

## 시작 전 확인

LiteLLM을 조직 공급자로 설정하려면 아래가 필요합니다.

**Caret 관리자 콘솔 접근 권한**  
조직 전체 설정을 적용하려면 관리자 권한이 필요합니다. [app.caret.bot](https://app.caret.bot)에서 **Settings → Caret Settings**로 이동할 수 있다면 권한이 있습니다.

<Info>
**빠른 확인**: 설정 페이지에서 공급자 구성 옵션이 보이면 정상입니다.
</Info>

**LiteLLM 프록시 실행 중**  
조직에서 접근 가능한 LiteLLM 프록시가 필요합니다(자체 호스팅 또는 클라우드).

<Note>
아직 LiteLLM을 배포하지 않았다면 인프라 팀과 협력해 프록시를 먼저 구축하세요.
</Note>

**LiteLLM 엔드포인트 정보**  
LiteLLM 프록시의 Base URL과(필요 시) Master Key가 필요합니다.

<Tip>
프록시가 개발 환경에서 접근 가능하며 필요한 모델이 활성화되어 있는지 확인하세요.
</Tip>

## 구성 단계

<Steps>
<Step title="Caret 설정 열기">
[app.caret.bot](https://app.caret.bot)에 관리자 계정으로 로그인한 뒤 **Settings → Caret Settings**로 이동합니다.

<Info>
올바른 관리자 권한이 있다면 공급자 구성 옵션이 표시됩니다.
</Info>
</Step>

<Step title="원격 공급자 구성 활성화">
**Enable settings** 토글을 켜서 원격 공급자 구성 옵션을 노출합니다. 이 설정으로 조직 전체에 공급자 구성을 적용할 수 있습니다.
</Step>

<Step title="API 공급자로 LiteLLM 선택">
**API Provider** 드롭다운에서 **LiteLLM**을 선택합니다. LiteLLM 구성 패널이 열립니다.
</Step>

<Step title="LiteLLM 설정 구성">
구성 패널에서 다음 값을 설정합니다.

<AccordionGroup>
<Accordion title="Base URL (필수)">
LiteLLM 프록시 엔드포인트 URL을 입력합니다. 예: `https://litellm.company.com` 또는 `http://your-proxy:4000`.

<Tip>
프로덕션에서는 HTTPS 사용을 권장합니다. 팀의 개발 환경에서 접근 가능한지 확인하세요.
</Tip>
</Accordion>

<Accordion title="Master Key (선택)">
프록시에 인증이 필요한 경우 Master Key를 입력합니다. 이 키는 조직 구성원 전체 요청에 사용됩니다.

<Note>
**중앙 API 키 관리**: 조직 레벨에서 Master Key를 설정하면 구성원은 개별 API 키를 관리할 필요가 없습니다.
</Note>

<Warning>
Master Key는 프록시에 대한 전체 접근 권한을 제공합니다. 인증이 필요한 경우에만 입력하세요.
</Warning>
</Accordion>
</AccordionGroup>
</Step>

<Step title="구성 저장">
설정이 끝나면 공급자 패널을 닫고 설정 페이지에서 **Save**를 클릭해 저장합니다.

저장 후 Caret 확장에 로그인한 모든 구성원이 LiteLLM을 사용하게 되며, 다른 공급자 선택이나 개인 Caret 계정 전환은 제한됩니다.

<Warning>
원격 구성이 활성화되면 구성원은 개인 Caret 계정으로 전환하거나 다른 조직에 참여할 수 없습니다. 이는 조직 단위 공급자 정책을 유지하기 위한 제한입니다.
</Warning>
</Step>
</Steps>

## 검증

다음 사항으로 구성을 확인하세요.

1. Enabled provider에 "LiteLLM" 표시 여부 확인
2. 페이지 새로고침 후 설정 유지 확인
3. 멤버 계정에서 LiteLLM만 보이는지 확인
4. 모델 드롭다운에 구성된 모델 표시 여부 확인

## 문제 해결

**구성원이 공급자를 보지 못함**  
설정을 저장했는지 확인하세요. 구성원이 올바른 조직에 속해 있는지, 프록시에 접근 가능한지 확인하세요.

**LiteLLM 프록시 연결 오류**  
Base URL이 정확한지, 방화벽/보안 그룹이 허용되어 있는지 확인하세요.

**인증 실패**  
Master Key를 사용하는 경우 키가 올바르고 권한이 있는지 확인하세요. LiteLLM 로그에서 인증 오류를 점검하세요.

**모델이 보이지 않음**  
LiteLLM 프록시에 모델이 올바르게 구성되어 있는지 확인하세요.

**설정 변경이 유지되지 않음**  
공급자 패널을 닫는 것만으로는 저장되지 않습니다. 반드시 설정 페이지의 Save 버튼을 클릭하세요.

**엔드포인트/키 변경 필요**  
언제든지 변경할 수 있으며, 변경 사항은 즉시 전체 구성원에게 적용됩니다.

자세한 정보는 [LiteLLM 문서](https://docs.litellm.ai/)를 참고하고 인프라 팀과 협의하세요.
