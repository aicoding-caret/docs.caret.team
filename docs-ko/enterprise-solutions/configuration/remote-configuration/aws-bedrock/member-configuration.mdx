팀 구성원은 로컬 개발 환경을 조직의 AWS Bedrock 설정과 연결할 수 있습니다. 관리자가 이미 공급자 설정을 완료했으므로, 본 가이드에서는 **자격 증명만 추가**해 바로 사용하도록 안내합니다.

## 시작 전 확인

조직의 AWS Bedrock 설정에 연결하려면 아래가 준비되어 있어야 합니다.

**Caret 확장 설치 및 조직 로그인**  
VS Code에 Caret 확장이 설치되어 있어야 하며, 조직 계정으로 로그인되어야 합니다. 아직 설치하지 않았다면 [설치 가이드](/getting-started/installing-caret)를 참고하세요.

<Info>
**빠른 확인**: VS Code에서 Caret 패널을 열고, 좌측 하단에 조직 이름이 표시되면 로그인된 상태입니다.
</Info>

**Bedrock 접근 권한이 있는 AWS 자격 증명**  
조직에서 설정한 리전에 Bedrock 접근 권한이 있는 AWS 자격 증명이 필요합니다.

<Note>
자격 증명이 없다면 IT/클라우드 팀에 문의해 Access Key 또는 AWS CLI 프로필을 발급받으세요.
</Note>

## 구성 단계

<Steps>
<Step title="Caret 설정 열기">
VS Code에서 다음 중 하나로 설정 패널을 엽니다.

- Caret 패널의 설정 아이콘(⚙️) 클릭
- 채팅 영역 바로 아래의 API Provider 드롭다운 클릭(예: `bedrock.anthropic.claude-sonnet-4-20250514-v1:0`)
</Step>

<Step title="인증 방식 선택">
AWS Bedrock에 연결할 인증 방식을 선택합니다.

<AccordionGroup>
<Accordion title="AWS Bedrock API Key">
Bedrock 전용 Access Key를 사용하는 방식입니다.

[AWS Bedrock API Keys 자세히 보기](https://docs.aws.amazon.com/bedrock/latest/userguide/api-keys.html)

1. **API Key** 라디오 버튼 선택
2. AWS Access Key ID와 Secret Access Key 입력
3. 자격 증명은 로컬에만 저장되며 VS Code 확장에서만 사용됩니다.
</Accordion>

<Accordion title="AWS Profile">
로컬 머신에 설정된 AWS CLI 프로필을 사용하는 방식입니다.

[AWS CLI Profiles 자세히 보기](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-profiles.html)

1. **AWS Profile** 라디오 버튼 선택
2. `~/.aws/credentials`에 등록된 프로필 이름 입력
3. Caret가 해당 프로필의 자격 증명을 사용합니다.
</Accordion>

<Accordion title="AWS Credentials">
환경 변수, EC2 역할 등 기본 AWS 자격 증명 체인을 사용하는 방식입니다.

1. **AWS Credentials** 라디오 버튼 선택
2. Caret가 표준 AWS credential provider chain으로 자격 증명을 자동 감지합니다.
</Accordion>
</AccordionGroup>

<Note>
AWS Region은 관리자가 미리 설정했으므로 확장에서 따로 입력하지 않아도 됩니다.
</Note>
</Step>

<Step title="구성 확인">
인증 방식을 선택하면 사용 가능한 기능에 체크 표시가 나타납니다.

- ✓ 이미지 지원
- ✓ 브라우저 사용
- ✓ 프롬프트 캐싱

크로스 리전 추론 및 글로벌 추론 프로필 같은 설정은 관리자가 제어하므로 잠금 아이콘(🔒)으로 표시됩니다.
</Step>

<Step title="연결 테스트">
Caret에 테스트 메시지를 보내 Bedrock 연결이 정상인지 확인합니다.

<Tip>
**테스트 권장**
실제 작업 전에 Plan 모드에서 테스트하면 안정적으로 확인할 수 있습니다.
</Tip>
</Step>
</Steps>

## 문제 해결

**인증 오류("Access Denied" 또는 "Invalid Credentials")**  
선택한 인증 방식이 Bedrock 호출 권한을 가지고 있는지 확인하세요. 필요한 IAM 권한은 `bedrock:InvokeModel` 및 `bedrock:InvokeModelWithResponseStream`입니다. 자세한 내용은 [AWS Bedrock IAM 권한](https://docs.aws.amazon.com/bedrock/latest/userguide/security-iam.html)을 참고하세요.

**리전 관련 오류 또는 "모델을 사용할 수 없음"**  
조직에서 설정한 리전을 관리자에게 확인하세요. 해당 리전에 대한 Bedrock 접근 권한이 필요합니다. [AWS Global Infrastructure](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/)를 참고하세요.

**AWS Bedrock 옵션이 보이지 않음**  
올바른 Caret 조직에 로그인되어 있는지 확인하세요. 관리자가 Bedrock 설정을 저장했는지 확인하고, 필요하면 로그아웃/로그인을 다시 시도하세요.

**AWS Credentials에서 자격 증명을 찾지 못함**  
AWS CLI가 설치되어 있는지 확인하고 `aws configure`로 자격 증명을 등록하세요([AWS CLI 설치 가이드](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)). `~/.aws/credentials` 파일에 자격 증명이 있는지 확인하세요. EC2/ECS 환경에서는 IAM 역할이 올바르게 부여되어 있어야 합니다. 환경 변수를 사용하는 경우 `AWS_ACCESS_KEY_ID`와 `AWS_SECRET_ACCESS_KEY`를 설정하세요.

## 보안 모범 사례

AWS 자격 증명 구성 시 아래를 권장합니다.

- 최소 권한 원칙의 IAM 역할 사용([AWS IAM 모범 사례](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html))
- API Key 사용 시 주기적으로 키 로테이션
- 자격 증명을 코드/버전 관리에 저장하지 않기
- 더 나은 관리성을 위해 AWS Profile 방식 선호
- 보안 강화를 위해 AWS SSO/연합 역할 사용 고려

조직 관리자가 허용한 모델만 표시되며, 선택 가능한 모델은 리전 설정에 따라 자동 반영됩니다. 자세한 정보는 [AWS Bedrock Model Access](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access.html)를 참고하세요.

추가 지원이 필요하면 [AWS Bedrock 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/what-is-bedrock.html)를 확인하고 조직의 클라우드 관리자와 협의하세요.
