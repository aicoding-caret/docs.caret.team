관리자는 호스팅된 관리자 콘솔에서 AWS Bedrock을 **조직 전체 LLM 공급자**로 설정할 수 있습니다. 이 방식은 Amazon 모델에 대한 일관된 접근을 보장하면서, 리전 제어와 기본 구성 옵션으로 보안/컴플라이언스 요구 사항을 충족합니다.

## 시작 전 확인

AWS Bedrock을 조직 공급자로 설정하기 전에 아래를 준비하세요.

**Caret 관리자 콘솔 접근 권한**  
조직 전체 설정을 적용하려면 관리자 권한이 필요합니다. [app.caret.bot](https://app.caret.bot)에서 **Settings → Caret Settings**로 이동할 수 있다면 권한이 있습니다.

**AWS Bedrock 계정 및 권한**  
AWS 계정에 Bedrock 관련 권한이 필요합니다.

<Note>
직접 AWS 접근 권한이 없다면 클라우드 팀과 협업해 필요한 권한을 먼저 준비하세요.
</Note>

**사용할 AWS 리전**  
조직 전체에 강제될 기본 리전을 결정하세요.

<Tip>
리전별로 사용 가능한 모델이 다를 수 있으니, 먼저 모델 가용성을 확인하세요.
</Tip>

## 구성 단계

<Steps>
<Step title="Caret 설정 열기">
[app.caret.bot](https://app.caret.bot)에 관리자 계정으로 로그인한 뒤 **Settings → Caret Settings**로 이동합니다.

<Info>
올바른 관리자 권한이 있다면 공급자 구성 옵션이 표시됩니다.
</Info>
</Step>

<Step title="원격 공급자 구성 활성화">
**Enable settings** 토글을 켜서 원격 공급자 구성 옵션을 노출합니다. 이 설정으로 조직 전체에 공급자 구성을 적용할 수 있습니다.
</Step>

<Step title="API 공급자로 AWS Bedrock 선택">
**API Provider** 드롭다운에서 **Amazon Bedrock**을 선택합니다. Bedrock 구성 패널이 열리며 조직 단위 설정을 구성할 수 있습니다.
</Step>

<Step title="Bedrock 설정 구성">
구성 패널에서 필요한 값을 설정합니다.

<AccordionGroup>
<Accordion title="Region (필수)">
`us-west-2` 또는 `us-east-1`과 같은 기본 리전을 입력합니다. 이 리전은 모든 조직 구성원에게 강제됩니다.

[AWS Global Infrastructure 보기](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/)

<Tip>
대부분의 조직은 모델 가용성이 높은 `us-east-1` 또는 `us-west-2`를 권장합니다.
</Tip>
</Accordion>

<Accordion title="Custom VPC Endpoint (선택)">
조직이 Bedrock 전용 VPC 엔드포인트를 사용한다면 여기에 입력해 모든 API 호출을 내부 네트워크로 라우팅할 수 있습니다.

[AWS PrivateLink 자세히 보기](https://docs.aws.amazon.com/vpc/latest/userguide/endpoint-services-overview.html)
</Accordion>

<Accordion title="Cross-region Inference (선택)">
기본 리전 용량이 부족할 때 다른 리전으로 자동 라우팅하도록 설정합니다. 수요가 높을 때 가용성을 유지하는 데 유용합니다.

[Inference Profiles 자세히 보기](https://docs.aws.amazon.com/bedrock/latest/userguide/inference-profiles-support.html)
</Accordion>

<Accordion title="Global Inference Profile (선택)">
AWS 글로벌 추론 라우팅을 사용해 가용성과 지연 시간을 기준으로 최적 리전에 자동 연결합니다.
</Accordion>

<Accordion title="Prompt Caching (선택)">
프롬프트 캐싱을 활성화하면 반복 요청에서 지연/비용을 줄일 수 있습니다.

[Prompt Caching 자세히 보기](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html)
</Accordion>
</AccordionGroup>
</Step>

<Step title="구성 저장">
설정이 끝나면 공급자 패널을 닫고 설정 페이지에서 **Save**를 클릭해 저장합니다.

저장 후에는 Caret 확장에 로그인한 모든 조직 구성원이 Bedrock을 사용하게 되며, 다른 공급자 선택이나 개인 Caret 계정 전환은 제한됩니다.

<Warning>
원격 구성이 활성화되면 구성원은 개인 Caret 계정으로 전환하거나 다른 조직에 참여할 수 없습니다. 이는 조직 단위 공급자 정책을 유지하기 위한 제한입니다.
</Warning>
</Step>
</Steps>

## 검증

다음 사항으로 구성을 확인하세요.

1. Enabled provider에 "Amazon Bedrock" 표시 여부 확인
2. 페이지 새로고침 후 설정 유지 확인
3. 멤버 계정으로 Bedrock만 보이는지 테스트

## 문제 해결

**구성원이 공급자를 보지 못함**  
설정을 저장했는지 확인하세요. 구성원 계정이 올바른 조직에 속해 있는지 점검하세요.

**설정 변경이 유지되지 않음**  
공급자 패널을 닫는 것만으로는 저장되지 않습니다. 반드시 설정 페이지의 Save 버튼을 클릭하세요.

**나중에 리전을 변경해야 함**  
리전은 언제든지 변경할 수 있습니다. 구성원은 변경된 리전에 접근 가능한 AWS 자격 증명을 갖고 있어야 합니다. 자세한 내용은 [AWS Bedrock 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/what-is-bedrock.html)를 참고하세요.

추가 정보는 [AWS Bedrock 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/what-is-bedrock.html)를 확인하고, 조직의 클라우드 담당자와 협의하세요.
