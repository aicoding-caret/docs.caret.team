팀 구성원은 로컬 개발 환경을 조직의 Google Vertex AI 설정에 연결할 수 있습니다. 관리자가 공급자 설정을 완료했으므로, **자격 증명만 추가**하면 됩니다.

## 시작 전 확인

조직의 Google Vertex AI 설정에 연결하려면 아래가 필요합니다.

**Caret 확장 설치 및 조직 로그인**  
VS Code에 Caret 확장이 설치되어 있어야 하며, 조직 계정으로 로그인되어야 합니다. 아직 설치하지 않았다면 [설치 가이드](/ko/getting-started/installing-caret)를 참고하세요.

<Info>
**빠른 확인**: VS Code에서 Caret 패널을 열고, 좌측 하단에 조직 이름이 표시되면 로그인된 상태입니다.
</Info>

**Vertex AI 접근 권한이 있는 Google Cloud 자격 증명**  
조직에서 설정한 프로젝트/리전에 대한 Vertex AI 접근 권한이 필요합니다.

<Note>
어떤 방식이 적합한지 모르겠다면 관리자 또는 IT 팀에 문의하세요.
</Note>

## 구성 단계

<Steps>
<Step title="Caret 설정 열기">
VS Code에서 다음 중 하나로 설정 패널을 엽니다.

- Caret 패널의 설정 아이콘(⚙️) 클릭
- 채팅 영역 바로 아래의 API Provider 드롭다운 클릭(예: `vertex_ai/gemini-pro`)

</Step>

<Step title="인증 방식 선택">
Google Vertex AI에 연결할 인증 방식을 선택합니다.

<AccordionGroup>
<Accordion title="Service Account Key">
서비스 계정 JSON 키 파일을 사용하는 방식입니다.

[Service Account Keys 자세히 보기](https://cloud.google.com/iam/docs/service-accounts)

1. **Service Account Key** 인증 방식 선택
2. 서비스 계정 JSON 키 파일 업로드 또는 붙여넣기
3. `aiplatform.user` 등 Vertex AI 권한이 필요
4. 자격 증명은 로컬에만 저장되며 VS Code 확장에서만 사용됩니다.
</Accordion>

<Accordion title="Google Cloud SDK">
로컬 머신에 설치된 Google Cloud SDK를 사용합니다.

[Google Cloud SDK 자세히 보기](https://cloud.google.com/sdk/docs/install)

1. **Google Cloud SDK** 인증 방식 선택
2. `gcloud auth login`으로 로그인
3. 조직의 Vertex AI 프로젝트 접근 권한 확인
4. Caret가 기본 Google Cloud 자격 증명을 자동 사용
</Accordion>

<Accordion title="Application Default Credentials">
Google Cloud의 ADC(Application Default Credentials) 체인을 사용합니다.

1. **Application Default Credentials** 방식 선택
2. 환경에 ADC가 올바르게 설정되어 있는지 확인
3. 중앙 관리되는 자격 증명 환경에 적합
4. Caret가 환경에서 자격 증명을 자동 감지
</Accordion>
</AccordionGroup>

<Note>
Google Cloud Project ID와 Region은 관리자가 이미 설정했으므로 확장에서 수정할 수 없습니다.
</Note>
</Step>

<Step title="구성 확인">
인증 방식을 선택하면 활성화된 기능이 체크 표시로 나타납니다.

- ✓ 이미지 지원(Gemini Pro Vision 등)
- ✓ 멀티모달 입력
- ✓ 함수 호출(지원 모델 한정)

프로젝트 ID와 리전은 관리자가 제어하므로 잠금 아이콘(🔒)으로 표시됩니다.
</Step>

<Step title="연결 테스트">
Caret에서 테스트 메시지를 보내 Vertex AI 프로젝트/리전과의 연결을 확인합니다.

<Tip>
**테스트 권장**
"안녕하세요" 같은 간단한 메시지로 연결을 확인한 뒤, 필요하면 이미지 입력도 테스트하세요.
</Tip>
</Step>
</Steps>

## 모델 사용

### 사용 가능한 모델군
조직의 Vertex AI 설정에 따라 다음 모델들이 제공됩니다.

**Gemini 모델:**
- **Gemini Pro**: 고급 추론, 코드 생성, 멀티모달
- **Gemini Pro Vision**: 이미지 이해 및 시각 질의응답
- **Gemini Ultra**: 복잡한 추론 작업용 최고 성능

**PaLM 모델:**
- **PaLM 2 for Text**: 텍스트 생성/완성
- **PaLM 2 for Chat**: 대화형 AI
- **Codey**: 코드 생성/설명 특화

**특수 모델:**
- **Text Embedding**: 시맨틱 검색/유사도 작업
- **Custom Models**: 조직의 파인튜닝 모델(가능한 경우)

### 모델 선택 전략

- **일반 작업**: Gemini Pro
- **이미지 작업**: Gemini Pro Vision
- **코드 중심 작업**: Codey 계열
- **복잡한 추론**: Gemini Ultra
- **임베딩**: Text Embedding

### 멀티모달 활용

- **이미지 분석**: Caret에 이미지를 업로드해 분석
- **시각 질의응답**: 이미지에 대한 질문 수행
- **코드 스크린샷**: 스크린샷 기반 코드 설명
- **문서 처리**: 차트/그래프 등 시각 데이터 분석

## 문제 해결

**Google Vertex AI 옵션이 보이지 않음**  
올바른 Caret 조직에 로그인되어 있는지 확인하세요. 관리자가 Vertex AI 구성을 저장했는지, 확장이 최신인지 점검하세요.

**인증 오류("Access Denied" 또는 "Invalid Credentials")**  
선택한 인증 방식이 설정된 프로젝트/리전에 대한 권한을 갖고 있는지 확인하세요. 필요한 IAM 권한은 `aiplatform.endpoints.predict`, `aiplatform.models.predict` 등입니다.

**프로젝트 접근 오류**  
관리자에게 조직에서 설정한 프로젝트를 확인하고, 해당 프로젝트 접근 권한이 있는지 점검하세요.

**리전 접근 오류**  
선택한 리전이 모델을 지원하는지 확인하세요. 일부 모델은 리전 제한이 있을 수 있습니다.

**Google Cloud SDK 인증 문제**  
SDK 설치 및 인증을 확인하세요.
```bash
gcloud auth login
gcloud config set project YOUR_PROJECT_ID
gcloud auth application-default login
```

**서비스 계정 키 오류**  
키가 유효하고 만료되지 않았는지 확인하세요. 서비스 계정에 Vertex AI 권한이 부여되어 있어야 합니다.

**모델 접근 오류 또는 "model not found"**  
조직 프로젝트/리전에 해당 모델이 활성화되어 있는지 확인하세요. 필요 시 관리자에게 문의하세요.

## 보안 모범 사례

Google Cloud 자격 증명 설정 시 다음을 권장합니다.

- 최소 권한의 서비스 계정 사용
- 서비스 계정 키 주기적 로테이션(권장: 90일)
- 자격 증명을 코드/버전 관리에 저장하지 않기
- 가능하면 Google Cloud SDK 사용
- 컨테이너 환경에서는 Workload Identity 고려
- 의심스러운 접근 발생 시 즉시 보고

조직 관리자가 허용한 모델과 리전만 표시됩니다. 프로젝트/리전 설정에 따라 사용 가능한 모델이 자동 반영됩니다.

자세한 인증/권한 정보는 [Google Cloud IAM 문서](https://cloud.google.com/iam/docs)를 참고하고 조직의 클라우드 관리자와 협의하세요.
