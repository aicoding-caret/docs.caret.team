## 어떤 구성이 적합한가요?

### SaaS 구성이 적합한 경우
- **빠른 셋업**이 필요함
- **중앙 관리**를 원함
- **표준 보안/컴플라이언스**만 필요함
- **소규모~중간 규모** 팀(수십~수백 명)

### Self-Hosted 구성이 적합한 경우
- **강화된 보안/컴플라이언스**가 필요함
- **기존 AWS/GCP 인프라**와 통합해야 함
- **커스텀 규칙/워크플로**가 필요함
- **대규모 조직**(수백~수천 명)
- **온프레미스/에어갭 환경**이 필요함

## 구성 비교

| 항목 | SaaS 구성 | Self-Hosted 구성 |
|---------|-------------------|---------------------------|
| **설정 난이도** | 쉬움 | 고급 |
| **배포 시간** | 수 분 | 수일~수주 |
| **필요 인프라** | 없음 | AWS/GCP/Azure |
| **컴플라이언스 기능** | 기본 | 고급 |
| **커스텀 규칙** | 불가 | 가능 |
| **멀티 계정 지원** | 불가 | 가능 |
| **VPC 통합** | 불가 | 가능 |
| **비용** | 낮음 | 높음 |

## 시작하기

<Steps>
<Step title="요구 사항 평가">
조직의 보안/컴플라이언스/인프라 요구 사항을 검토해 적합한 구성을 결정합니다.
</Step>

<Step title="구성 경로 선택">
SaaS 구성(간단) 또는 Self-Hosted 구성(고급)을 선택합니다. 필요하다면 [배포 가이드](/enterprise-solutions/configuration/choosing-your-deployment)를 참고하세요.
</Step>

<Step title="구성 가이드 진행">
선택한 구성 유형의 상세 가이드를 따라 설정을 완료합니다.
</Step>

<Step title="팀 온보딩">
설정이 완료되면 팀원들은 공급자별 가이드를 따라 연결할 수 있습니다.
</Step>
</Steps>
