YOLO 모드는 Caret가 **모든 작업을 자동 승인**하며 완전 자율로 동작하게 합니다. 이 문서는 엔터프라이즈 관리자가 조직 전체에서 YOLO 모드 사용을 제어하는 방법을 설명합니다.

<Note>
YOLO 모드의 기능/위험/모범 사례는 [기능 문서](/ko/features/yolo-mode)에서 확인하세요.
</Note>

## 개요

YOLO 모드가 활성화되면 파일 변경, 터미널 명령, 브라우저 동작, 모드 전환까지 **모든 작업이 자동 승인**됩니다. 속도는 극대화되지만 안전 장치가 사라집니다.

<Warning>
YOLO 모드는 강력하지만 잠재적으로 위험합니다. 어떤 팀/사용자에게 허용할지 신중히 결정해야 합니다.
</Warning>

## 엔터프라이즈 관리자 구성

관리자는 원격 구성으로 조직 구성원이 YOLO 모드를 활성화할 수 있는지 제어할 수 있습니다.

### 모든 사용자에 대해 YOLO 모드 비활성화

원격 구성 JSON에 다음을 추가합니다.

```json
{
  "yoloModeAllowed": false
}
```

`yoloModeAllowed`가 `false`이면:
- 모든 UI에서 YOLO 모드 토글이 비활성화됩니다.
- 사용자는 로컬 설정에서도 YOLO 모드를 켤 수 없습니다.
- 정책이 즉시 전체 구성원에게 적용됩니다.
- 조직 정책이 개인 설정보다 우선합니다.

### 모든 사용자에 대해 YOLO 모드 허용

```json
{
  "yoloModeAllowed": true
}
```

`yoloModeAllowed`가 `true`이거나 생략되면:
- 사용자가 로컬 Caret 설정에서 YOLO 모드를 켜고 끌 수 있습니다.
- 조직 차원의 제한이 적용되지 않습니다.

- 자동화 테스트 시나리오
- 데모/교육 환경

## 엔터프라이즈 고려 사항

### 보안 영향

YOLO 모드가 활성화되면:

**위험 요소**
- 모든 도구 실행이 자동 승인됨
- 다수 파일에 실수가 빠르게 전파될 가능성
- 보안 취약점을 사전에 잡기 어려움
- 기존 보안 제어를 우회할 위험

**완화책**
- 포괄적인 로깅/모니터링 구축
- 비프로덕션 환경에 제한
- 정기 보안 리뷰 필수
- 버전 관리/롤백 절차 확립

### 모니터링 요구 사항

YOLO 모드 허용 시 다음을 권장합니다.

**필수 모니터링**
1. **실시간 활동 추적**
   - 누가 YOLO 모드를 켰는지 기록
   - YOLO 모드 활성 여부 추적
   - 모든 자동화 작업 로그

2. **감사 추적 유지**
   - YOLO 모드 세션 전체 기록
   - 자동화된 작업 내역 저장
   - 컴플라이언스 목적 기록 유지

3. **이상 탐지**
   - 비정상 패턴 경보
   - 고위험 작업 자동 감지
   - 잠재적 보안 사고 모니터링

### YOLO 모드 사용 추적

YOLO 모드가 활성화된 경우 다음을 추적하세요.

**텔레메트리 이벤트**
- YOLO 모드 토글 이벤트 기록
- YOLO 모드 활성 상태에서 실행된 작업 기록
- 조직 전체 사용 통계 제공

**작업 히스토리**
- 작업 메타데이터에 YOLO 모드 상태 기록
- 자동 승인된 동작 로그 포함
- 사후 리뷰 가능

**감사 로그**
- 모든 자동 승인 결정 기록
- 도구 실행 시각 기록
- 규제 환경을 위한 감사 추적 제공

## 기술적 구현

### 구성 관리

**원격 구성 예시:**

```json
{
  "yoloModeAllowed": false
}
```

이 설정은:
- 모든 클라이언트에 즉시 적용
- 개인 설정으로 override 불가
- Caret 재시작 이후에도 유지
- 조직 구성원 전체에 동기화됨

### 정책 적용 흐름

1. 사용자가 엔터프라이즈 구성 서버에 인증
2. 원격 구성이 다운로드되어 적용됨
3. 로컬 UI가 정책을 반영
4. 정책이 금지하면 YOLO 토글이 비활성화됨
5. 사용자에게 제한 사유 안내

## 컴플라이언스 고려 사항

규제가 있는 조직에서는:

**SOC 2**
- YOLO 모드는 변경 관리 통제와 충돌할 수 있음
- 허용/차단 여부를 보안 정책에 문서화
- 허용 시 보완 통제 적용

**GDPR/데이터 보호**
- 자동화 작업이 데이터 처리 정책을 위반하지 않도록 관리
- 감사 로그로 자동 처리 기록 유지

**산업별 규제**
- 금융: 규제 요구와 충돌 가능
- 의료: HIPAA 감사 요건 위반 가능
- 공공기관: 승인 워크플로 요구와 상충
