## 공급자 구성의 의미

Caret에서 공급자 구성은 다음을 가능하게 합니다.

1. **자격 증명 중앙 관리**: API 키와 인증 정보를 한 곳에서 관리
2. **모델 접근 제어**: 팀이 사용할 수 있는 모델 정의
3. **공급자 사용 강제**: 승인된 공급자만 사용하도록 제한

## 동작 방식

공급자 설정은 원격 구성 JSON으로 정의됩니다.

```json
{
  "providerSettings": {
    "provider": "bedrock",
    "bedrockRegion": "us-east-1",
    "bedrockServiceRole": "arn:aws:iam::..."
  }
}
```

설정이 적용되면:
- 모든 구성원에게 자동 적용
- 개인 설정을 덮어씀
- 조직 전체에서 동일한 공급자 정책 유지

## 구성 옵션

### 공급자 선택

지원되는 공급자:
- **bedrock**: AWS Bedrock
- **vertex**: Google Vertex AI
- **openai**: OpenAI API
- **azure**: Azure OpenAI
- **litellm**: LiteLLM 프록시

### 인증 방식

공급자별 지원 인증:

**AWS Bedrock**
- 크로스 계정 IAM 역할
- Access Key(프로덕션 비권장)

**Google Vertex AI**
- 서비스 계정 JSON 키
- Workload Identity(GKE)

**OpenAI/Azure**
- API 키

**LiteLLM**
- 엔드포인트 URL + API 키

## 예시 구성

### AWS Bedrock + IAM 역할
```json
{
  "providerSettings": {
    "provider": "bedrock",
    "bedrockRegion": "us-east-1",
    "bedrockServiceRole": "arn:aws:iam::123456789012:role/CaretBedrockRole"
  }
}
```

### Google Vertex AI
```json
{
  "providerSettings": {
    "provider": "vertex",
    "vertexProject": "my-project-id",
    "vertexRegion": "us-central1"
  }
}
```

### LiteLLM 프록시
```json
{
  "providerSettings": {
    "provider": "litellm",
    "litellmBaseUrl": "https://litellm.company.com",
    "litellmApiKey": "sk-..."
  }
}
```
