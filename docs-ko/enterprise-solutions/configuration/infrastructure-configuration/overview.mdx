<Warning>
**Self-Hosted 구성 경로**

이 섹션은 **Self-Hosted Caret 인프라**를 배포하는 엔터프라이즈를 위한 문서입니다. 복잡한 보안, 컴플라이언스, 멀티 환경 요구 사항을 다루며 YAML, Kubernetes/Helm, IaC 기반 구성을 사용합니다.

**간단한 설정이 필요하신가요?** [SaaS 공급자 구성](/ko/enterprise-solutions/configuration/remote-configuration/overview)에서 app.caret.bot 관리자 콘솔을 통해 빠르게 구성할 수 있습니다.
</Warning>

Self-Hosted 구성은 AI 공급자부터 커스텀 워크플로까지, 조직의 Caret 배포를 인프라 레벨에서 제어할 수 있게 해줍니다. 이 섹션에서는 고급 보안/컴플라이언스/운영 기능을 포함한 엔터프라이즈 배포 구성 방법을 설명합니다.

## 시작하기

1. **요구 사항 점검**: 인프라와 통합 요구 사항 검토
2. **공급자 설정**: 엔터프라이즈 자격 증명으로 AI 공급자 구성
3. **보안 설정**: 규칙 및 접근 제어 적용
4. **모니터링 설정**: 텔레메트리/모니터링 활성화
5. **사용자 온보딩**: 개발 팀에 구성 배포

## 엔터프라이즈 아키텍처 고려사항

### 보안 & 컴플라이언스
- **제로 트러스트 모델**: 모든 구성은 제로 트러스트 보안 모델을 지원
- **감사 로깅**: 모든 구성 변경에 대한 감사 추적
- **역할 기반 접근**: 관리자 역할별 세밀한 권한
- **데이터 주권**: 민감 데이터가 인프라 경계 안에 머물도록 보장

### 확장성 & 성능
- **멀티 리전 지원**: 여러 지역에 구성 배포
- **로드 밸런싱**: 여러 엔드포인트로 추론 요청 분산
- **캐싱 전략**: 지능형 캐시로 성능 최적화
- **속도 제한**: 구성 가능한 레이트 리밋

### 통합 & 자동화
- **GitOps 통합**: 구성 변경을 코드로 관리
- **CI/CD 파이프라인 연계**: 구성 배포 자동화
- **Webhook 지원**: 구성 변경 이벤트에 자동화 트리거
- **API 중심 설계**: 모든 구성을 API로 관리

## 구성 관리 기능

모든 엔터프라이즈 구성은 다음을 지원합니다.

- **버전 관리**: 변경 이력 추적
- **환경 승격**: dev → staging → production 배포
- **롤백 기능**: 문제 있는 구성 빠르게 복구
- **구성 검증**: 변경 사항 자동 테스트
- **드리프트 감지**: 구성 드리프트 모니터링
