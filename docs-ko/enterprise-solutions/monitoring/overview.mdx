## Caret 텔레메트리

Caret는 익명 사용량을 수집하는 옵트인 텔레메트리를 제공합니다.

- 기능 사용 패턴
- 작업 완료율
- 오류 발생률
- 성능 지표

사용자는 설정에서 텔레메트리를 켜거나 끌 수 있습니다. 수집 데이터는 익명이며 코드, 파일 경로, 민감 정보는 포함되지 않습니다.

자세한 설정은 [Caret 텔레메트리](/ko/enterprise-solutions/monitoring/telemetry)를 참고하세요.

## OpenTelemetry 통합

고급 모니터링이 필요하다면 OpenTelemetry의 OTLP(OpenTelemetry Protocol)로 메트릭과 로그를 자체 인프라로 내보낼 수 있습니다.

이를 통해 다음이 가능합니다.
- 기존 관측성 플랫폼으로 텔레메트리 전송
- Datadog, New Relic, Grafana Cloud 등과 통합
- 모니터링 데이터에 대한 완전한 통제 유지
- 조직 전체 지표 집계

<Note>
OpenTelemetry 통합은 **선택 사항**이며 추가 구성이 필요합니다. 대부분의 사용자는 기본 텔레메트리만으로 충분합니다.
</Note>

설정 방법은 [OpenTelemetry](/ko/enterprise-solutions/monitoring/opentelemetry)를 참고하세요.

## 사용 시나리오

### Caret 텔레메트리를 사용할 때
- 익명 사용 데이터로 Caret 개선에 기여하고 싶을 때
- 추가 설정 없이 빠르게 시작하고 싶을 때
- 대부분의 일반 사용자/팀

### OpenTelemetry를 사용할 때
- 자체 시스템에 정밀한 지표가 필요할 때
- 기존 관측성 인프라와 통합해야 할 때
- 디버깅용 상세 로그/메트릭이 필요할 때
- 커스텀 대시보드/알림을 구성할 때

## 시작하기

<Steps>
<Step title="접근 방식 선택">

</Step>

<Step title="텔레메트리 활성화">
기본 텔레메트리는 설정에서 활성화하고, OpenTelemetry는 별도 가이드를 따릅니다.
</Step>

<Step title="수집 확인">
텔레메트리가 정상 수집되는지 확인합니다.
</Step>
</Steps>
