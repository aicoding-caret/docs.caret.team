<Note>
Caret 기준 문서입니다. 계정/조직 구성, 캐럿 전용 규칙/프롬프트 표준화에 따라 수집 이벤트가 달라질 수 있습니다. 세부 수집 항목은 [원격 측정](/ko/more-info/telemetry) 문서와 `docs/merging/caret-policy-reference.md`를 참고하세요.
</Note>

Caret는 사용 패턴을 이해하고 제품을 개선하기 위해 텔레메트리를 제공합니다. 사용자는 공유 여부를 직접 제어할 수 있습니다.

## Caret 텔레메트리란?

텔레메트리는 다음과 같은 **익명 사용 이벤트**를 수집합니다.

- 사용된 기능(도구, 커맨드, 워크플로)
- 작업 완료율
- 오류 발생 빈도
- 성능 지표

<Info>
수집 데이터는 **익명화**되며 코드 내용, 파일 내용, 민감 정보는 포함하지 않습니다.
</Info>

## 사용자 제어

### 텔레메트리 켜기/끄기

사용자는 Caret 설정에서 텔레메트리를 제어할 수 있습니다.

1. Caret 설정 열기
2. "Caret 텔레메트리" 토글 찾기
3. 필요에 따라 활성/비활성

변경 사항은 즉시 적용됩니다.

### 수집되는 항목

텔레메트리가 활성화되면 다음이 수집됩니다.

<AccordionGroup>
  <Accordion title="기능 사용" icon="cursor-click">
    - 실행된 도구(예: read_file, execute_command)
    - 슬래시 커맨드 사용
    - 워크플로 트리거
    - 설정 변경
  </Accordion>
  
  <Accordion title="작업 지표" icon="tasks">
    - 작업 시작/완료 이벤트
    - Plan/Act 모드 전환
    - 체크포인트 사용
    - 작업 지속 시간
  </Accordion>
  
  <Accordion title="오류 이벤트" icon="triangle-exclamation">
    - API 실패
    - 도구 실행 오류
    - 시스템 오류
    - 오류 유형 및 빈도
  </Accordion>
</AccordionGroup>

### 수집되지 않는 항목

Caret 텔레메트리는 다음을 **수집하지 않습니다**.

- 코드 또는 파일 내용
- 파일 경로나 이름
- 명령 인자/파라미터
- 대화 내용
- 개인정보
- API 키/자격 증명

## 엔터프라이즈 구성

관리자는 원격 구성으로 기본 텔레메트리 상태를 설정할 수 있습니다.

```json
{
  "telemetryEnabled": true
}
```

<Note>
조직 정책이 설정되어 있더라도 사용자는 로컬 설정에서 텔레메트리를 비활성화할 수 있습니다.
</Note>

## 고급 모니터링

상세 모니터링이 필요한 조직은 OpenTelemetry 통합을 통해 자체 관측성 시스템으로 텔레메트리를 내보낼 수 있습니다.

[엔터프라이즈 모니터링](/ko/enterprise-solutions/monitoring/overview) 문서에서 선택지를 확인하세요.

## 텔레메트리의 가치

익명 사용 데이터는 다음을 개선하는 데 사용됩니다.

- **버그 식별**: 사용자에게 영향을 주는 문제 파악
- **기능 우선순위**: 실제 사용량 기반 로드맵 조정
- **성능 개선**: 느린 작업 발견 및 최적화
- **신뢰성 향상**: 오류율 추적 및 감소
