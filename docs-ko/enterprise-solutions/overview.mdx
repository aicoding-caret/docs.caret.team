Caret Enterprise는 수많은 개발자가 이미 사용하는 오픈 소스 아키텍처 위에 **중앙집중형 거버넌스**를 제공합니다. 코드는 사용자의 환경에 그대로 남고, 기존에 계약한 추론 비용을 그대로 사용하면서도 조직 단위 배포에 필요한 보안과 관측성 기능을 확보할 수 있습니다.

## 제공 가치

운영 팀이 프로덕션 배포에 필요한 다섯 가지 핵심 역량을 제공합니다. 각각은 조직 규모로 AI 코딩을 확장할 때 필요한 요구 사항을 해결합니다.

### 설계 단계부터 보안

코드는 환경을 벗어나지 않습니다. Caret는 모든 처리를 로컬에서 수행하며, 업로드/인덱싱/학습에 데이터를 사용하지 않습니다.

### Bring Your Own Inference

기존 클라우드 계약과 협상된 요금을 그대로 사용합니다. 대부분의 AI 도구는 추론 비용에 마진을 붙이지만, Caret는 공급자에 직접 연결합니다.

연결 가능한 공급자:
- AWS Bedrock
- Google Vertex AI
- Azure OpenAI
- Anthropic direct
- OpenAI direct
- Cerebras
- OpenAI 호환 엔드포인트

새 모델이 출시되면 즉시 교체할 수 있습니다. Claude Sonnet 4.5를 기본 모델로 쓰고, GPT-5는 복잡한 리팩터링에 활용하며, 오픈 소스 모델은 간단한 작업에 사용할 수 있습니다. 기존 클라우드 크레딧과 스타트업 프로그램 계약을 그대로 활용할 수 있습니다. Caret는 에이전트 루프를 처리하고, 조직은 추론을 담당합니다. 마진도, 벤더 종속도 없습니다.

### 대규모 거버넌스

수천 명의 개발자가 AI를 사용할 때는 중앙 통제가 필수입니다. 개별 API 키가 각자의 노트북에 흩어지면 보안 위험과 비용 이슈가 발생합니다.

Enterprise 거버넌스는 다음을 제공합니다:
- **SSO 인증**: 개인 키 대신 기업 계정 사용
- **역할 기반 접근 제어**: 팀/프로젝트 단위 세밀한 권한
- **모델/도구 제어**: 팀별로 허용 모델 및 도구 관리
- **원격 구성**: 대시보드에서 전체 설정 관리
- **전체 감사 로그**: AI 상호작용을 상세히 기록

한 번 설정하면 조직 전체에 배포할 수 있습니다. 개발자는 선호하는 방식대로 일하고, 플랫폼 팀은 통제권을 유지합니다.

### 완전한 관측성

기존 관측성 스택으로 로그를 내보내 사용량, 비용, 성능을 추적합니다.

- **OpenTelemetry 내보내기**: Datadog, Grafana, Splunk 등과 연동
- **실시간 분석**: 채택/성능/패턴 추적
- **비용 세분화**: 팀과 모델별 비용 확인
- **JSON 출력**: 기존 대시보드와 통합 가능

프로덕션 시스템에 요구하는 관측성 기준을 그대로 적용할 수 있습니다.

## 배포

Caret Enterprise는 인프라에 안전하게 연결됩니다. 클라우드, 온프레미스, 에어갭 환경 모두에서 동작하며, 기존 보안 정책과 컴플라이언스 요구 사항에 맞게 구성할 수 있습니다.

조직 배포 흐름:
1. Caret Core를 인프라에 연결
2. SSO, RBAC, 거버넌스 정책 설정
3. 기존 소프트웨어 배포 체계를 통해 개발자에게 배포
4. 관측성 도구로 사용량 모니터링
