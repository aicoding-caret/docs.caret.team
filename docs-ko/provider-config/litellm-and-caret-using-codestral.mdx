### <BrandName particle="and" /> 함께 LiteLLM 사용

이 가이드는 Codestral 모델을 사용해 LiteLLM 데모를 구동하고 <BrandName particle="and" /> 연결하는 과정을 보여줍니다.

#### 전제 조건

-   LiteLLM 이미지를 로컬에서 실행할 수 있는 [Docker CLI 또는 Docker Desktop](https://www.docker.com/get-started/)
-   예시 구성에 사용할 Codestral API 키 (Mistral API 키와 별도)

#### 설정

1. **`.env` 파일 생성 후 값 입력**

    ```bash
    # 팁: 다음 명령으로 임의의 영숫자 키를 생성할 수 있습니다.
    # openssl rand -base64 32 | tr -dc 'A-Za-z0-9' | head -c 32
    LITELLM_MASTER_KEY=YOUR_LITELLM_MASTER_KEY
    CODESTRAL_API_KEY=YOUR_CODESTRAL_API_KEY
    ```

    _참고: localhost 범위로 제한되더라도 LITELLM_MASTER_KEY는 안전한 값으로 설정하는 것이 좋습니다._

2. **구성 파일 생성**

    LiteLLM 구성을 담을 `config.yaml` 파일을 준비합니다. 여기서는 `codestral-latest` 한 개 모델을 `codestral`로 등록합니다.

    ```yaml
    model_list:
        - model_name: codestral
          litellm_params:
              model: codestral/codestral-latest
              api_key: os.environ/CODESTRAL_API_KEY
    ```

#### 데모 실행

1. **LiteLLM 도커 컨테이너 실행**

    ```bash
    docker run \
        --env-file .env \
        -v $(pwd)/config.yaml:/app/config.yaml \
        -p 127.0.0.1:4000:4000 \
        ghcr.io/berriai/litellm:main-latest \
        --config /app/config.yaml --detailed_debug
    ```

2. **<BrandName particle="from" /> LiteLLM 설정**

    LiteLLM 서버가 올라오면 <BrandName /> 설정에 다음을 입력합니다:

    - Base URL: `http://0.0.0.0:4000/v1`
    - API Key: `.env`의 `LITELLM_MASTER_KEY`
    - Model ID: `codestral` 또는 `config.yaml`에서 지정한 이름
