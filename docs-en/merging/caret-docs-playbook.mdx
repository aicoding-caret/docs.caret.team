---

> 다음 업데이트 시, 이 문서만 따라도 캐럿 전용 정책과 브랜딩을 유지한 채 최신 Caret 변경을 반영할 수 있도록 작성했습니다.

## 0. 준비
- 기준 브랜치: `caret-main`(현재)와 `caret-latest`(업스트림 동기화) 확인.
- 명령어: `git diff caret-main..caret-latest --name-only -- docs`로 영향 파일 목록 확보.
- 정책 수집: 캐럿 전용 모델 지원/차단, 인증·라우팅, 브랜딩/아이콘, 텔레메트리 정책을 최신화.

## 1. 대상 선정
- 우선순위: 온보딩/소개 → CLI → 모델·프로바이더 → 핵심 기능 → 엔터프라이즈/정책.
- 새로 추가된 파일부터 처리(`git diff --stat` 참고) 후 기존 파일의 삽입/삭제를 검토.

## 2. 콘텐츠 반영
- Caret 변경을 그대로 옮기되, 캐럿 전용 차이는 `<Note>`로 “캐럿 전용” 표기(모델 제한, 라우팅, UI 차이 등).
- 모델/프로바이더: 지원/비지원 모델, 지역/계정 제한, 인증 경로를 명확히 기재.
- 온보딩/CLI: 지원 OS/빌드/설치 경로, 버튼/라벨 등 브랜딩 차이 병기.

## 3. 네비 & 메타
- 새 문서가 있으면 `docs/docs.json`의 적절한 그룹에 추가(Reference 또는 해당 섹션).
- 리디렉션 필요 시 `redirects`에 추가. 스타일 영향 시 `docs/styles.css` 확인.

## 4. 검증
- 내부 링크/에셋 경로 확인(`rg "(\\.\\./|/assets)" docs/<path>` 정도로 빠른 점검).
- 로컬 프리뷰: `npm run docs` 등 문서 빌드로 렌더/링크 깨짐 확인.

## 5. 기록
- 워크로그에 처리/미처리/TODO를 남기고, 남은 문서와 필요한 정책 정보를 명시.
- 캐럿 정책 변화가 있으면 이 플레이번 문서와 `update-strategy.mdx`를 함께 업데이트.
